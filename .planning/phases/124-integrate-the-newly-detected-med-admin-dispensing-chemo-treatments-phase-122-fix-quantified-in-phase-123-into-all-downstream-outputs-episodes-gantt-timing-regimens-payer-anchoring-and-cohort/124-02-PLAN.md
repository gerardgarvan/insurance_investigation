<objective>
Consume the R/26 `source_hints` column (Plan 01) so that R/28 populates the Gantt
Source Table (D-05) and Code Type (D-04) columns correctly for the newly-surfaced
DISPENSING and MED_ADMIN chemo records, and fix the pre-existing R/20 MED_ADMIN
code_type mislabel (RESEARCH Pitfall 8 / Open Question 4).

The core problem (RESEARCH Critical Finding): R/28 builds source_table/code_type by
exploding triggering_codes and joining a CODE-keyed xlsx lookup where every chemo_rxnorm
code maps to source_table="PRESCRIBING" (utils_xlsx_lookups.R line 80). The same RxCUI
can arrive from PRESCRIBING, DISPENSING, or MED_ADMIN, so the code-keyed lookup cannot
distinguish physical table. The fix is to override the lookup result with the per-code
source_hints R/26 now carries.

Purpose: without this override, DISPENSING/MED_ADMIN records silently show as PRESCRIBING
in gantt_episodes.csv, and D-05 is unmet.
Output: modified R/28_episode_classification.R and R/20_treatment_inventory.R.
Structurally grep-verifiable on Windows; runtime confirmed in Plan 04.
</objective>
