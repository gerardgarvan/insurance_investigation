<objective>
Add the source-provenance and raw-drug-name plumbing that the Phase 122 chemo fix needs
in order to label the newly-surfaced DISPENSING and MED_ADMIN records correctly downstream.

This is the NET-NEW code that must land BEFORE any regeneration produces correct output
labels. Two coupled changes, both scoped to chemo only:

1. Carry a per-code `source_hint` (physical table name) from R/26's stacking step through
   to a new `source_hints` column on `treatment_episodes.rds`, parallel to the existing
   `triggering_codes` column. This is the mechanism R/28 (Plan 02) will use to populate the
   Gantt Source Table column with DISPENSING / MED_ADMIN (D-05) instead of the wrong
   "PRESCRIBING" the code-keyed xlsx lookup returns.
2. Let `get_chemo_hits()` optionally return `RAW_MEDADMIN_MED_NAME`, and wire a
   canonicalized raw-name fallback tier into R/26 Section 5B so MED_ADMIN drugs that miss
   MEDICATION_LOOKUP + RxNorm cache still resolve to a canonical spelling (D-06/D-07).

Purpose: without source_hints, DISPENSING/MED_ADMIN records mislabel as PRESCRIBING in the
Gantt (RESEARCH Pitfall 2 / Critical Finding). Without the raw-name fallback, MED_ADMIN
free-text drugs would drop out of drug_names.

Output: modified R/utils/utils_treatment.R and R/26_treatment_episodes.R. Structurally
grep-verifiable on Windows; runtime confirmed in Plan 04 on HiPerGator.
