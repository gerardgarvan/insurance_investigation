<critical_domain_notes>
**RUCA taxonomy (from RESEARCH.md, VERIFIED from USDA ERS):**
- Primary codes are integers 1-10 (decimal RUCA codes like 1.0, 4.1, 10.6 have primary = floor(code))
- Primary 1-3 = Metropolitan; 4-6 = Micropolitan; 7-9 = Small town; 10 = Rural
- Code 99 = "Not coded" (water-only / zero-population ZIPs) -> maps to NA (NOT Rural)
- Unmatched ZIPs (blank, ZIP+4, non-numeric, out-of-range) -> NA

**File structure caveats (from RESEARCH.md Open Questions):**
- The 2020 ZIP RUCA xlsx may have a title row or codebook sheet before the data sheet
- Column headers may vary ("ZIP_CODE" vs "ZIPCODE" vs long descriptive strings)
- Robust approach: read the file, inspect names once with a `message(paste(names(x), collapse=", "))`, then read column 1 and column 2 by position: `select(ZIP_CODE = 1, RUCA_code = 2)`
- Try sheet = 1 first; if the first sheet looks like a codebook (row 1 is descriptive text, not headers), use `sheet = "Data"` or the appropriate named sheet

**ZIP normalization (from RESEARCH.md Pitfall 1 & 2):**
- DEMOGRAPHIC.ZIP_CODE is loaded as `col_character()` (R/01 line 204) so leading zeros are preserved
- May contain 9-digit ZIP+4 ("32606-1234" or "326061234"), whitespace, or non-numeric text
- Sequence: `str_trim -> str_sub(1,5) -> str_pad(5, pad="0") -> if_else(str_detect("^[0-9]{5}$"), ., NA)`

**Grain confusion (from RESEARCH.md Pitfall 5):**
Sheet 1 is patient-level; Sheets 2-4 are encounter-level. High-utilizer patients (many encounters) dominate encounter counts. Label each sheet title clearly.

**SECTION 3 -- ruca_tier_label() FUNCTION:**
```r
# Map decimal RUCA code to 4-tier condensed rurality label
# See RESEARCH.md 'RUCA Code Taxonomy' -- primary code = floor(decimal code)
ruca_tier_label <- function(ruca_code) {
  primary <- floor(ruca_code)
  dplyr::case_when(
    primary %in% 1:3  ~ "Metropolitan",
    primary %in% 4:6  ~ "Micropolitan",
    primary %in% 7:9  ~ "Small town",
    primary == 10     ~ "Rural",
    TRUE              ~ NA_character_    # 99 (not coded), NA, unexpected -> Unknown
  )
}
```

```r
build_crosstab <- function(df, row_col, col_col) {
  # Counts rows of df by (row_col, col_col), pivots wide, sorts both axes
  # ascending alphabetically, and appends row-total and column-total.
  # Result: a tibble with the row_col as the first column, one column per
  # distinct col_col value (alphabetical), and a Total column + Total row.

  tidy <- df %>%
    count(.data[[row_col]], .data[[col_col]], name = "n")

  col_levels  <- sort(unique(tidy[[col_col]]))
  row_levels  <- sort(unique(tidy[[row_col]]))

  wide <- tidy %>%
    pivot_wider(names_from = all_of(col_col),
                values_from = n,
                values_fill = 0L) %>%
    select(all_of(row_col), all_of(col_levels))

  wide <- wide %>% arrange(match(.data[[row_col]], row_levels))

  # Row totals
  wide <- wide %>%
    rowwise() %>%
    mutate(Total = sum(c_across(where(is.numeric)), na.rm = TRUE)) %>%
    ungroup()

  # Column totals row (label 'Total' in the row_col column)
  totals_row <- wide %>%
    summarise(!!row_col := "Total",
              across(where(is.numeric), sum))

  bind_rows(wide, totals_row)
}
```
