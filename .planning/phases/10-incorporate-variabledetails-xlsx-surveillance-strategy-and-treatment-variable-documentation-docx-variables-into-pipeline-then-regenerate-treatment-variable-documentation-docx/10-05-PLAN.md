4. Build markdown content as a character vector, section by section. The documentation should cover per D-16:

**Section 1: Title and metadata**
```
# PCORnet Hodgkin Lymphoma Pipeline -- Variable Documentation
**Generated:** {Sys.Date()}
**Pipeline version:** Phase 10
**Data source:** PCORnet CDM v6.0+ (Mailhot HL Cohort Extract)
```

**Section 2: Cohort Definition**
- Describe filter chain: DEMOGRAPHIC -> has_hodgkin_diagnosis -> with_enrollment_period -> exclude_missing_payer
- List HL ICD-10 codes (length of ICD10_HL_CODES) and ICD-9 codes (length of ICD9_HL_CODES)
- Mention ICD-O-3 histology code range (9650-9667) for tumor registry
- HL_SOURCE: DIAGNOSIS only, TR only, Both
- HL_VERIFIED: 1 if any HL evidence, 0 if excluded as "Neither"

**Section 3: Demographics and Enrollment**
- SEX, RACE, HISPANIC from DEMOGRAPHIC
- age_at_enr_start, age_at_enr_end from ENROLLMENT
- enrollment_duration_days
- first_hl_dx_date

**Section 4: Payer Variables**
- PAYER_CATEGORY_PRIMARY: Mode of 9-category payer across all encounters
- PAYER_CATEGORY_AT_FIRST_DX: Payer at first HL diagnosis encounter
- DUAL_ELIGIBLE: 0/1 flag for Medicare+Medicaid evidence
- PAYER_TRANSITION: 0/1 flag for payer category change over time
- N_ENCOUNTERS, N_ENCOUNTERS_WITH_PAYER

**Section 5: Treatment Variables**
For each treatment type (Chemotherapy, Radiation, Stem Cell Transplant):
- HAD_{TYPE}: 0/1 flag
- FIRST_{TYPE}_DATE: Date of first treatment
- Detection sources: list all sources (PROCEDURES CPT/HCPCS/ICD-10-PCS/ICD-9-CM, PRESCRIBING, DISPENSING RXNORM, MED_ADMIN RXNORM, DIAGNOSIS evidence codes, ENCOUNTER DRG codes, PROCEDURES revenue codes, TUMOR_REGISTRY dates)
- Code counts: read from TREATMENT_CODES list dynamically, e.g., "Chemotherapy HCPCS: {length(TREATMENT_CODES$chemo_hcpcs)} codes"
- List actual code ranges (first few + "..." if many)

**Section 6: Treatment-Anchored Payer Variables**
- PAYER_AT_CHEMO, PAYER_AT_RADIATION, PAYER_AT_SCT
- Window: +-30 days from first treatment date
- Computation: Mode of payer category across encounters in window

**Section 7: Timing Variables (D-12)**
- DAYS_DX_TO_CHEMO: Integer days from first_hl_dx_date to FIRST_CHEMO_DATE
- DAYS_DX_TO_RADIATION: Same for radiation
- DAYS_DX_TO_SCT: Same for SCT

**Section 8: Surveillance Modalities (D-01, D-03, D-04)**
For each modality, document:
- HAD_{MODALITY}, FIRST_{MODALITY}_DATE, N_{MODALITY}
- Source table(s): PROCEDURES (CPT/HCPCS) and/or LAB_RESULT_CM (LOINC)
- Code counts from SURVEILLANCE_CODES/LAB_CODES dynamically
- Detection window: Post-diagnosis (after first_hl_dx_date)

List of modalities from SURVEILLANCE_CODES names, dynamically. Include:
- Mammogram, Breast MRI, Echocardiogram, Stress Test, ECG, MUGA, PFT, TSH, CBC

**Section 9: Lab Results (D-02)**
For each lab type:
- HAD_{LAB}, FIRST_{LAB}_DATE, N_{LAB}
- Source: LAB_RESULT_CM via LOINC code matching
- Code counts from LAB_CODES dynamically
- Detection window: Post-diagnosis

List: CRP, ALT, AST, ALP, GGT, Bilirubin, Platelets, FOBT

**Section 10: Survivorship Encounter Classification (D-05 through D-10)**
- Level 1 (ENC_NONACUTE_CARE): AV+TH encounters post-diagnosis
- Level 2 (ENC_CANCER_RELATED): Level 1 + HL DX on encounter
- Level 3 (ENC_CANCER_PROVIDER): Level 2 + provider is oncology specialty
  - Provider specialties: list from PROVIDER_SPECIALTIES$cancer_oncology
- Level 4 (ENC_SURVIVORSHIP): Level 3 + personal history ICD codes on encounter
  - ICD-9 codes: list from SURVIVORSHIP_CODES$personal_history_icd9
  - ICD-10 codes: list from SURVIVORSHIP_CODES$personal_history_icd10
- Each level: HAD_, N_, FIRST_*_DATE

**Section 11: Pending / Deferred Variables**
- "Surveillance Strategy - Pending" modalities (10 items without codes)
- Adriamycin/doxorubicin variables (deferred pending SEER*Rx)
- NDC-based matching (deferred)
- treatment_type_post_dx factor (individual HAD_* flags used instead)
