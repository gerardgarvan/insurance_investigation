# Phase 10: Incorporate VariableDetails.xlsx Surveillance Strategy and Treatment_Variable_Documentation.docx Variables into Pipeline, then Regenerate Documentation - Context

**Gathered:** 2026-03-31
**Status:** Ready for planning

<domain>
## Phase Boundary

This phase adds three capability sets to the R pipeline: (1) post-diagnosis surveillance modality detection from VariableDetails.xlsx (9 modalities + labs), (2) survivorship encounter classification from VariableDetails.xlsx (4 levels), (3) timing derivation variables from Treatment_Variable_Documentation.docx, and (4) auto-generated comprehensive variable documentation covering the full pipeline output. No new treatment types or treatment intensity variables are added. NDC-based matching remains deferred. Doxorubicin-specific variables (adriamycin_6mo, n_dox_encounters) are deferred.

</domain>

<decisions>
## Implementation Decisions

### Surveillance Modality Scope
- **D-01:** Implement all 9 surveillance modalities from VariableDetails.xlsx "Surveillance Strategy" sheet: Mammogram, Breast MRI, Echocardiogram, Stress test, Electrocardiogram, MUGA, Pulmonary function test, Thyroid stimulating hormone, Complete blood count.
- **D-02:** Include labs from the "Labs" sheet: CRP, liver function tests (ALT, AST, ALP, GGT, bilirubin), platelets, APRI index, FOBT. Requires loading LAB_RESULT_CM table with LOINC code matching.
- **D-03:** Surveillance detection is restricted to post-diagnosis (after first HL diagnosis date), not post-treatment.
- **D-04:** Each modality gets three output columns: HAD_{MODALITY} (0/1 flag), FIRST_{MODALITY}_DATE (date), N_{MODALITY} (count of events post-diagnosis).

### Survivorship Encounter Classification
- **D-05:** Load PROVIDER table into 01_load_pcornet.R to enable PROVIDER_SPECIALTY_PRIM matching for encounter levels 3 and 4.
- **D-06:** Implement all 4 encounter levels as separate per-patient binary flags: ENC_NONACUTE_CARE (level 1: AV+TH encounters), ENC_CANCER_RELATED (level 2: AV+TH with HL DX on encounter), ENC_CANCER_PROVIDER (level 3: level 2 + provider is hematologist/oncologist/surgical oncologist/radiation oncologist), ENC_SURVIVORSHIP (level 4: level 3 + personal history of treatment ICD codes).
- **D-07:** Cancer-related visit criterion uses HL-specific codes only (C81.x, 201.x), NOT any reportable cancer.
- **D-08:** Per-patient summary: binary flags + counts + first dates for each level, added to hl_cohort.
- **D-09:** Level 4 personal history ICD codes: V87.41, V87.42, V87.43, V87.46, V15.3 (ICD-9) and Z92.21, Z92.22, Z92.23, Z92.25, Z92.3 (ICD-10).
- **D-10:** Cancer provider specialties from NUCC taxonomy: hematologist, oncologist, hematologist/oncologist, surgical oncologist, radiation oncologist (match against PROVIDER_SPECIALTY_PRIM).

### Missing Docx Variables
- **D-11:** Skip treatment_type_post_dx (4-level factor) -- individual HAD_* flags are sufficient.
- **D-12:** Add timing derivation columns: DAYS_DX_TO_CHEMO, DAYS_DX_TO_RADIATION, DAYS_DX_TO_SCT computed from existing FIRST_*_DATE minus first HL diagnosis date.
- **D-13:** Skip Adriamycin/doxorubicin variables (adriamycin_6mo, any_doxorubicin_6mo, n_dox_encounters) -- requires SEER*Rx NDC crosswalk, deferred.
- **D-14:** Keep RXNORM-only matching for DISPENSING/MED_ADMIN (no NDC matching added).

### Documentation Generation
- **D-15:** Regenerate Treatment_Variable_Documentation as both markdown (.md source of truth) and docx (for sharing). Auto-generate from pipeline code.
- **D-16:** Documentation covers full pipeline reference: treatment variables, surveillance modalities, labs, survivorship encounters, payer variables, cohort definition, timing variables.
- **D-17:** Auto-generated by an R script that reads 00_config.R code lists, function signatures, and pipeline structure programmatically. Stays in sync with code changes.
- **D-18:** Methodology only -- no patient-level counts or run statistics in the documentation.

### Claude's Discretion
- LAB_RESULT_CM col_types specification (researcher determines from PCORnet CDM spec)
- PROVIDER table col_types specification
- Internal structure of surveillance detection functions (new script or extend existing)
- How to match NUCC taxonomy provider specialties (exact match vs pattern)
- R package choice for docx generation (officer, officedown, or rmarkdown)
- Markdown template structure and section ordering
- Whether surveillance code lists go in 00_config.R or a separate surveillance config
- How to handle the "Surveillance Strategy - Pending" items (10 modalities without codes)

</decisions>

<code_context>
## Existing Code Insights

### Reusable Assets
- `has_chemo()`, `has_radiation()`, `has_sct()` in 03_cohort_predicates.R: Pattern for detection functions returning tibble(ID, HAD_*) -- surveillance modality functions should follow same pattern extended with date and count columns.
- `compute_payer_mode_in_window()` in 10_treatment_payer.R: Generic window-based payer computation, template for date-anchored analysis.
- `TREATMENT_CODES` in 00_config.R: Named list structure for code lists. Surveillance codes can follow same pattern (SURVEILLANCE_CODES list).
- 11_generate_pptx.R: Existing R-based document generation using officer package -- can be reused/extended for docx documentation output.
- `encounters` tibble from 02_harmonize_payer.R: Already has ENC_TYPE, ADMIT_DATE, payer_category per encounter -- extend with survivorship level classification.
- `pcornet$DIAGNOSIS` in 04_build_cohort.R: Already loaded, can be joined to encounters for DX_type criterion.

### Established Patterns
- Detection functions return tibble(ID, HAD_*) with integer 0/1 flags via union of patient IDs from multiple sources
- First dates computed as min(date) grouped by ID
- PX_TYPE matching: "CH" for CPT/HCPCS, "09" for ICD-9-CM, "10" for ICD-10-PCS, "RE" for revenue codes
- Null-safe table access: `if (!is.null(pcornet$TABLE)) { ... }`
- Source logging via message(glue("...")) with formatted counts

### Integration Points
- 01_load_pcornet.R: Add LAB_RESULT_CM and PROVIDER to PCORNET_TABLES vector and col_types specifications
- 00_config.R: Add SURVEILLANCE_CODES list with code vectors per modality, LAB_CODES list with LOINC codes, SURVIVORSHIP_CODES for personal history ICD codes, PROVIDER_SPECIALTIES for cancer provider matching
- 04_build_cohort.R: Add surveillance flags, survivorship encounter flags, and timing derivation columns to final cohort assembly
- New script (e.g., 13_surveillance.R): Surveillance modality detection functions
- New script (e.g., 14_survivorship_encounters.R): Survivorship encounter classification functions
- New script (e.g., 15_generate_documentation.R): Auto-documentation generation

</code_context>

<specifics>
## Specific Ideas

- VariableDetails.xlsx "Surveillance Strategy" sheet is the authoritative source for surveillance modality codes (CPT, HCPCS, ICD-10-PCS, ICD-10, LOINC per modality).
- VariableDetails.xlsx "Labs" sheet is the authoritative source for lab test codes (CPT, LOINC per lab).
- VariableDetails.xlsx "Survivorship Encounter" sheet defines the 4-level encounter classification logic.
- VariableDetails.xlsx "Treatment" sheet (123 rows) contains treatment codes that may supplement existing TREATMENT_CODES -- researcher should cross-reference with current config to identify any missing codes.
- Documentation auto-generation should be a standalone R script that can be re-run whenever the pipeline changes.
- The "Surveillance Strategy - Pending" sheet lists 10 modalities without codes yet (chemistry, dental, colonoscopy, etc.) -- note these in documentation as "pending future addition" but don't implement.

</specifics>
