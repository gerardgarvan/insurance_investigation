Create R/14_survivorship_encounters.R implementing D-05 through D-10:

1. Header comment block explaining the 4-level classification hierarchy and decision references.

2. Library imports: dplyr, glue.

3. classify_survivorship_encounters(post_dx_date_map) function that returns:
   tibble(ID, HAD_ENC_NONACUTE, N_ENC_NONACUTE, FIRST_ENC_NONACUTE_DATE,
                HAD_ENC_CANCER_RELATED, N_ENC_CANCER_RELATED, FIRST_ENC_CANCER_RELATED_DATE,
                HAD_ENC_CANCER_PROVIDER, N_ENC_CANCER_PROVIDER, FIRST_ENC_CANCER_PROVIDER_DATE,
                HAD_ENC_SURVIVORSHIP, N_ENC_SURVIVORSHIP, FIRST_ENC_SURVIVORSHIP_DATE)

Implementation steps within the function:

a. **Level 1: Non-acute care encounters (ENC_NONACUTE_CARE)**
   - Filter pcornet$ENCOUNTER for ENC_TYPE %in% c("AV", "TH") (ambulatory visit, telehealth)
   - inner_join with post_dx_date_map to restrict to cohort patients
   - Filter ADMIT_DATE > first_hl_dx_date (post-diagnosis only, D-03 principle)
   - Group by ID: count, min date

b. **Level 2: Cancer-related visits (ENC_CANCER_RELATED)**
   - Start from Level 1 encounters (subset of AV+TH post-diagnosis)
   - inner_join with pcornet$DIAGNOSIS on ENCOUNTERID (same encounter has HL DX)
   - Filter DX for HL codes only (D-07: C81.x ICD-10, 201.x ICD-9) using:
     ```r
     filter(
       (DX_TYPE == "10" & DX %in% ICD10_HL_CODES) |
       (DX_TYPE == "09" & DX %in% ICD9_HL_CODES)
     )
     ```
   - Use normalize_icd_code() from utils_icd.R if needed for dot handling
   - Group by ID: count, min date

c. **Level 3: Cancer provider visits (ENC_CANCER_PROVIDER)**
   - Start from Level 2 encounters
   - Null-safe check: if pcornet$PROVIDER is NULL, set Level 3/4 to 0 for all patients and log warning
   - left_join to pcornet$PROVIDER on PROVIDERID (left_join because PROVIDERID may be NULL per Pitfall 2 in RESEARCH.md)
   - Filter for PROVIDER_SPECIALTY_PRIM %in% PROVIDER_SPECIALTIES$cancer_oncology
   - Log count of encounters with NULL PROVIDERID for transparency
   - Group by ID: count, min date

d. **Level 4: Survivorship visits (ENC_SURVIVORSHIP)**
   - Start from Level 3 encounters
   - Check if the same encounter (ENCOUNTERID) has a personal history ICD code in DIAGNOSIS:
     ```r
     # Personal history codes on the encounter
     has_personal_hx <- pcornet$DIAGNOSIS %>%
       filter(
         (DX_TYPE == "09" & DX %in% SURVIVORSHIP_CODES$personal_history_icd9) |
         (DX_TYPE == "10" & DX %in% SURVIVORSHIP_CODES$personal_history_icd10)
       ) %>%
       distinct(ENCOUNTERID)

     level4_encounters <- level3_encounters %>%
       semi_join(has_personal_hx, by = "ENCOUNTERID")
     ```
   - Group by ID: count, min date

e. **Assembly:** For each level, create per-patient summary with HAD_ (0/1), N_ (count), FIRST_*_DATE. Left_join all to base post_dx_date_map IDs so every cohort patient gets a row (with 0/NA for non-matches).

f. **Logging:** For each level, log count of patients with at least one encounter at that level:
```r
message(glue("[Survivorship] Level 1 (Non-acute): {n1} patients"))
message(glue("[Survivorship] Level 2 (Cancer-related): {n2} patients"))
message(glue("[Survivorship] Level 3 (Cancer provider): {n3} patients"))
message(glue("[Survivorship] Level 4 (Survivorship): {n4} patients"))
```

IMPORTANT:
- D-07: Cancer-related uses HL codes ONLY (C81.x, 201.x), not all cancer codes
- D-09: Personal history codes span ICD-9 AND ICD-10, use DX_TYPE filter
- D-10: NUCC codes for provider matching (exact %in%, not regex)
- Pitfall 2: PROVIDERID may be NULL in many ENCOUNTER rows -- use left_join, do not inner_join
- Pitfall 4: Personal history codes cross ICD eras -- filter by DX_TYPE
