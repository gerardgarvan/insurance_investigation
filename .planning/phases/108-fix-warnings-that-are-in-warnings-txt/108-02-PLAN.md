min_or_na <- function(x, na.rm = TRUE) {
  if (all(is.na(x))) return(NA)
  min(x, na.rm = na.rm)
}

}

**Investigate and fix the TABLE-2 vs TABLE-1 row count issue (per D-09)**

The current sanity check at lines 361-366 is:
```r
if (nrow(table2) >= nrow(table1)) {
  warning("[R/36 WARNING] TABLE-2 row count >= TABLE-1 -- expected TABLE-2 (chemo-only) to be smaller")
}
```

**Root cause analysis:** TABLE-2 has one row per encounter+medication combination (per D-06: "one row per encounter+medication"). TABLE-1 has one row per encounter+treatment_type combination (via distinct()). A single chemo encounter can have MULTIPLE medications (e.g., ABVD regimen = 4 drugs), so TABLE-2 naturally has more rows than TABLE-1 for chemotherapy encounters, even though TABLE-2 only covers chemo encounters.

**The sanity check assumption is wrong.** TABLE-2 is NOT a row-count subset of TABLE-1 because the grain is different:
- TABLE-1 grain: `(patient, encounter, treatment_type)` — one row per encounter
- TABLE-2 grain: `(patient, encounter, medication)` — multiple rows per encounter

**Fix:** Replace the row-count comparison with a meaningful encounter-level subset check. TABLE-2's unique ENCOUNTERIDs should be a subset of TABLE-1's ENCOUNTERIDs (all chemo encounters in TABLE-2 should appear in TABLE-1).

Replace lines 361-366 with:
```r
# Sanity check: TABLE-2 encounters should be a subset of TABLE-1 encounters
# Note: TABLE-2 may have MORE rows than TABLE-1 because TABLE-2 grain is
# per-encounter+medication (multiple drugs per encounter) while TABLE-1
# grain is per-encounter+treatment_type (one row per encounter).
t2_encounters <- unique(table2$ENCOUNTERID)
t1_encounters <- unique(table1$ENCOUNTERID)
t2_not_in_t1 <- setdiff(t2_encounters, t1_encounters)
if (length(t2_not_in_t1) > 0) {
  warning(glue("[R/36 WARNING] {length(t2_not_in_t1)} TABLE-2 encounters not found in TABLE-1 -- data consistency issue"))
} else {
  message(glue("  Sanity check PASSED: all {length(t2_encounters)} TABLE-2 encounters found in TABLE-1 ({length(t1_encounters)} total encounters)"))
}
```

This correctly validates the containment relationship at the encounter level, not the row level.
