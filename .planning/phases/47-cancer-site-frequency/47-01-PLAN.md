Create R/47_cancer_site_frequency.R following the standalone diagnostic script pattern from R/45_radiation_cpt_audit.R. The script has 6 sections:

**SECTION 1: SETUP**
- suppressPackageStartupMessages for dplyr, stringr, glue, openxlsx2, readxl, purrr

- message("=== Phase 47: Cancer Site Frequency ===")

**SECTION 2: LOAD AND PARSE CancerSiteCategories.xlsx**
- groups_raw <- read_excel("CancerSiteCategories.xlsx", sheet = "Groups")
- groups_df <- groups_raw %>% select(category = 1, icd10 = 2, icdo3 = 3) -- use positional to avoid column name ambiguity
- Verify nrow(groups_df) == 42 with a stopifnot and message
- Log how many categories have non-NA icd10 and icdo3 cells

**SECTION 3: RANGE EXPANSION FUNCTIONS**
Define two local helper functions:

`expand_icd_token(token)`:
- str_trim the token
- If contains "-" (range): split on "-", str_trim each part
- Extract prefix (everything before trailing digits) with str_extract(x, "^.*?(?=\\d+$)") and numeric suffix with str_extract(x, "\\d+$")
- This handles C7A010 -> prefix "C7A0", suffix "10" and C000 -> prefix "C", suffix "000"
- Apply toupper to prefix
- Compute width from nchar of start numeric suffix for zero-padding
- Generate paste0(prefix, formatC(start_n:end_n, width=width, flag="0"))
- Apply normalize_icd() to result
- If numeric extraction returns NA or prefixes differ between start and end, warn and return both endpoints as individual codes
- If no "-" (single code): normalize_icd(token)

`expand_code_string(code_str)`:
- If is.na(code_str) or trimmed empty, return character(0)
- Split on "," with str_split
- Apply expand_icd_token to each token via unlist(lapply(...))
- Return unique normalized code vector

**SECTION 4: BUILD PREFIX LOOKUP (first match wins)**
- Iterate categories in spreadsheet order (1 to 42)
- For each category, expand icd10 codes and icdo3 codes separately
- Build two named vectors (or tibbles): icd10_prefix_to_cat and icdo3_prefix_to_cat
- For icd10: for each expanded code, if not already in the lookup, assign it to this category index
- For icdo3: same logic, but SKIP categories where all expanded codes are in 8000-9999 range (morphology codes, not topography -- per phase decision). Check: if all codes after removing "C" prefix are >= 8000, skip. If codes start with "C" and the numeric part is < 8000, they are topography -- keep them.
- Log: "Built ICD-10 prefix lookup: {N} unique prefixes across 42 categories"
- Log: "Built ICD-O-3 prefix lookup: {N} unique prefixes across {M} categories (skipped {K} morphology-only categories)"

**SECTION 5: QUERY DATA AND COUNT**

ICD-10 from DIAGNOSIS:
- diagnosis_icd10 <- get_pcornet_table("DIAGNOSIS") %>% filter(DX_TYPE == "10") %>% select(ID, DX, ENCOUNTERID) %>% materialize() %>% mutate(DX_norm = normalize_icd(DX))
- Log row count: "Loaded {nrow} ICD-10 DIAGNOSIS rows"

ICD-O-3 from TUMOR_REGISTRY_ALL:
- tr_topo <- get_pcornet_table("TUMOR_REGISTRY_ALL") %>% select(ID, topo_raw = coalesce(TOPOGRAPHY_CODE, ICDOSITE)) %>% filter(!is.na(topo_raw)) %>% materialize() %>% mutate(topo_norm = normalize_icd(topo_raw))
- Log: "Loaded {nrow} TUMOR_REGISTRY rows with topography codes"
- Also log: head(unique(tr_topo$topo_norm), 10) to verify format at runtime (addresses open question about C-prefix vs bare digits)

Matching -- use the prefix lookup approach:
For each category i (1:42):
  - Get the expanded ICD-10 prefix codes for category i
  - Filter diagnosis_icd10 where any prefix in the expanded set is a prefix of DX_norm (use startsWith). Implement via: `dx_matches <- diagnosis_icd10 %>% filter(purrr::map_lgl(DX_norm, ~any(startsWith(.x, icd10_codes_i))))`
  - icd10_patients_i <- n_distinct(dx_matches$ID)
  - icd10_encounters_i <- n_distinct(dx_matches$ENCOUNTERID)
  - Collect matched ICD-10 patient IDs for combined unique count
  - Same for ICD-O-3: tr_matches using icdo3 prefix codes against topo_norm
  - icdo3_patients_i <- n_distinct(tr_matches$ID)
  - icdo3_records_i <- nrow(tr_matches)
  - Collect matched ICD-O-3 patient IDs
  - combined_patients_i <- n_distinct(c(dx_matches$ID, tr_matches$ID))

Build result_df tibble with columns: category, icd10_patients, icd10_encounters, icdo3_patients, icdo3_records, combined_patients -- one row per category in spreadsheet order.

**Totals row (Claude's discretion):**
- Compute sum of each numeric column for per-category totals
- For combined_patients total: compute n_distinct of ALL matched patient IDs across both sources and all categories (the true unique count)
- Add totals row with category = "TOTAL" to result_df
- Log summary: "Total unique patients across all cancer sites: {N}"

**SECTION 6: WRITE STYLED XLSX**
Follow the openxlsx2 pattern from R/45_radiation_cpt_audit.R exactly:

Data rows (row 3 onward):

Totals row (row 3+42 = row 45):
