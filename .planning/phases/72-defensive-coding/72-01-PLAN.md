The file must contain these 5 functions per D-13:

1. `assert_rds_exists(path, script_name)` -- Per SAFE-01. Uses `checkmate::assert_file_exists(path, access = "r", .var.name = glue::glue("[{script_name} ERROR] Expected RDS file"))`. Returns `invisible(path)`.

2. `assert_df_valid(df, name, required_cols, script_name, allow_empty = FALSE)` -- Per SAFE-02. Two checks: (a) `checkmate::assert_data_frame(df, min.rows = if (allow_empty) 0 else 1, .var.name = glue::glue("[{script_name} ERROR] {name}"))` and (b) `checkmate::assert_names(colnames(df), must.include = required_cols, .var.name = glue::glue("[{script_name} ERROR] {name} columns"))`. Returns `invisible(df)`.

3. `assert_col_types(df, type_spec, script_name)` -- Per D-08. `type_spec` is a named list like `list(ID = "character", date_col = "Date", count = "integer")`. Loop over names(type_spec), check column exists with `if (!col_name %in% colnames(df)) stop(glue(...))`. For each type check: Date uses `inherits(df[[col_name]], "Date")`, character uses `is.character()`, integer/numeric uses `is.numeric()`. Error messages follow `[{script_name} ERROR] Column '{col_name}' must be {expected_class}, got {actual_class} -- {fix_hint}` pattern. Returns `invisible(df)`.

4. `warn_date_range(df, col, lo, hi, script_name)` -- Per D-09, D-10. Guard: if col not in colnames or not Date class, emit warning and return early. Filter non-NA values outside [lo, hi]. If any found, emit `warning(glue("[{script_name} WARNING] {n_out} dates outside {lo} to {hi} range in '{col}' -- range: {range_actual[1]} to {range_actual[2]} -- tumor registry may include pre-2012 dates (legitimate)"))`. Uses `dplyr::filter(!is.na(.data[[col]]) & (.data[[col]] < lo | .data[[col]] > hi))`. Returns `invisible(df)`.

5. `warn_row_count(df, name, min_expected = NULL, max_expected = NULL, script_name)` -- Per D-10. Check `nrow(df)` against bounds. If below min_expected: `warning(glue("[{script_name} WARNING] {name} has {actual} rows, expected >= {min_expected} -- possible data loss from join/filter"))`. If above max_expected: `warning(glue("[{script_name} WARNING] {name} has {actual} rows, expected <= {max_expected} -- possible cartesian product from join"))`. Returns `invisible(df)`.

All functions must:
- Use `glue::glue()` for message interpolation (SAFE-03)
- Follow `[R/XX ACTION] message` format per D-11, D-12
- Use magrittr pipe `%>%` where pipes are needed (lintr compliance)
- Keep lines under 150 characters (lintr compliance)
- Include roxygen2-style `#'` documentation for each function
