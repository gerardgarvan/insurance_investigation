  <action>
Create standalone diagnostic script R/22_multi_source_overlap_detection.R following the established pattern from R/21_all_site_duplicate_dates.R. The script has 7 sections:

**Header block:** Phase 25 comment header with requirements (SAMEDT-01 through SAMEWK-03), purpose, output file list, usage, dependencies. Same format as R/21.

**Libraries:** dplyr, lubridate, glue, readr, stringr, tidyr. Source R/00_config.R. Conditional source R/01_load_pcornet.R if pcornet does not exist.

**SECTION 1 — Load and prepare encounters:**
- Load pcornet$ENCOUNTER. Do NOT join DEMOGRAPHIC — use ENCOUNTER.SOURCE directly (per user decision: no DEMOGRAPHIC.SOURCE for site assignment).
- Parse ADMIT_DATE via as.Date(format="%Y-%m-%d") with parse_pcornet_date() fallback if parse rate < 50% (same pattern as R/21 Section 3).
- Log total encounters, parse rate, unique ENCOUNTER.SOURCE values found.

**SECTION 2 — Same-date multi-source detection (SAMEDT-01):**
- Group encounters by ID + admit_date_parsed. For each group, compute n_sources = n_distinct(SOURCE). Keep only groups where n_sources > 1.
- For each multi-source patient-date: record ID, ADMIT_DATE, n_sources, n_encounters, source_combo (sorted concatenated with "+" separator, e.g., "AMS+FLM+UFH"), sources_list (comma-separated for readability).
- Store as `same_date_detail` data frame.

**SECTION 3 — Per-ENCOUNTER.SOURCE summary for same-date (SAMEDT-02):**
- For each unique ENCOUNTER.SOURCE value, compute: total_encounters (all encounters from this source), n_multi_source_dates (patient-dates where this source appears alongside another source), n_patients_affected (distinct patients with at least one multi-source date involving this source), pct_encounters_overlapping (approximate: n encounters on multi-source dates / total encounters for that source).
- Store as `per_source_same_date` data frame.
- Console output: table of per-source counts.

**SECTION 4 — Source combination frequencies for same-date (SAMEDT-03):**
- From same_date_detail, count frequency of each source_combo string. Report both n_patient_dates (number of patient-date pairs) and n_total_encounters (sum of encounters across those pairs).
- Store as `same_date_combo_freq`.
- Console: top 10 combinations ranked by frequency.

**SECTION 5 — Same-week near-duplicate detection (SAMEWK-01, SAMEWK-02):**
- For each patient, find pairs of encounters from DIFFERENT sources where ADMIT_DATE differs by 1-7 calendar days (abs(date1 - date2) between 1 and 7 inclusive). Same-date pairs are excluded (they are captured in Section 2).
- Implementation approach: self-join encounters on ID, filter where SOURCE_x != SOURCE_y and abs(admit_date_x - admit_date_y) is in 1:7. Deduplicate pairs (keep where SOURCE_x < SOURCE_y or date_x <= date_y to avoid double-counting).
- For each near-duplicate pair: ID, admit_date_1, source_1, admit_date_2, source_2, day_gap, source_combo (sorted "+" format).
- Store as `same_week_detail`.
- IMPORTANT: Pairwise only — no transitive chaining. If patient has encounters on days 1, 4, 10 from different sources, pairs are (1,4) only; (4,10) is a separate pair; (1,10) is excluded (gap > 7).

**SECTION 6 — Per-ENCOUNTER.SOURCE summary and combo frequencies for same-week (SAMEWK-02, SAMEWK-03):**
- Per-source summary: n_near_duplicate_pairs involving this source, n_patients_affected.
- Combo frequencies: count of each source_combo, same format as Section 4.
- Console: per-source counts table + top 10 same-week combinations.
- Console: side-by-side comparison of same-date vs same-week rates per source.

**SECTION 7 — Write CSV outputs and console summary:**
4 CSV files to output/tables/ with `multi_source_` prefix:

1. `multi_source_same_date_detail.csv` — One row per patient-date: ID, ADMIT_DATE, n_sources, n_encounters, source_combo, sources_list. Ordered by source_combo, then ID, then date.

2. `multi_source_same_week_detail.csv` — One row per near-duplicate pair: ID, admit_date_1, source_1, admit_date_2, source_2, day_gap, source_combo. Ordered by source_combo, then ID.

3. `multi_source_combo_frequencies.csv` — Two sections stacked: same-date combos then same-week combos. Columns: match_type ("same_date" or "same_week"), source_combo, n_patient_dates (or n_pairs for same-week), n_total_encounters, rank. Order by match_type then desc(count).

4. `multi_source_per_source_summary.csv` — One row per ENCOUNTER.SOURCE value. Columns: SOURCE, total_encounters, n_same_date_multi_source_dates, n_same_date_patients_affected, pct_same_date_overlapping, n_same_week_near_dup_pairs, n_same_week_patients_affected. Order by SOURCE.

Final console summary with strrep('=', 70) borders:
- Total encounters analyzed, total unique patients, ADMIT_DATE parse rate
- Same-date: total multi-source patient-dates, total patients affected
- Same-week: total near-duplicate pairs, total patients affected
- Per-source one-liner with same-date count + same-week count
- Top 10 source combos for each category
- List of 4 CSV files written
- Phase 26 note: "These outputs feed R/23_overlap_classification.R for field-by-field comparison"

Apply HIPAA suppression to CSV counts: replace values 1-10 with "<11" in count columns. Use a simple helper (inline, not imported) that converts count to character and masks small cells. Keep raw numeric values in console output (not persisted).
