## Summary

Phase 12 refines an existing 20-slide PowerPoint deck generated by R's officer package. The work involves three categories of changes: (1) adding definitions via glossary slide and per-slide footnotes, (2) fixing graph issues (payer consolidation, masked date filtering, label clipping, overflow bin), and (3) slide removals and additions. All changes build on established patterns in `11_generate_pptx.R` and `16_encounter_analysis.R`.

**Primary recommendation:** Use `ph_with()` + `fpar()` for slide footnotes, `flextable::footnote()` for table footnotes (only if needed), mutate + `geom_histogram()` for overflow bins, `filter(DX_YEAR != 1900)` for masked dates, and `coord_cartesian(clip = "off")` + `ylim()` expansion for label cutoff. All techniques verified against officer 0.7.3, flextable 0.9.10 (Feb 2026), and ggplot2 4.0.1+.

## Architecture Patterns

### Pattern 1: Adding Footnote Text to Slide Bottom
**What:** Use `ph_with()` + `fpar()` + `ftext()` to place small text at the bottom of a slide, below tables or images.

**When to use:** Per-slide term definitions (D-02), exclusion counts (D-11).

**Example:**
```r
# Source: https://davidgohel.github.io/officer/reference/ph_with.html
# Add footnote text at bottom of slide (5.2 inches from top for 5.625" slide height)
pptx <- pptx %>%
  ph_with(
    value = fpar(ftext("*Primary Insurance = most prevalent payer across all encounters",
                       prop = fp_text(font.size = 8, italic = TRUE,
                                      font.family = "Calibri", color = "#666666"))),
    location = ph_location(left = 0.5, top = 5.0, width = 9, height = 0.4)
  )
```

**Key insight:** Footnotes are NOT automatic in officer — must manually position via `ph_location()`. Use `top = 5.0-5.2` for 5.625" slide height to place at bottom.

### Pattern 2: Creating Histogram Overflow Bins
**What:** Mutate data to cap values at a threshold and create a labeled ">500" category, then use `geom_histogram()` + `geom_text()` to visually distinguish overflow bin.

**When to use:** Histogram with long tail where extreme values distort scale (D-08).

**Example:**
```r
# Source: https://edwinth.github.io/blog/outlier-bin/ (adapted)
# Mutate to create overflow bin category
hist_data <- hist_data %>%
  mutate(
    N_ENC_BINNED = if_else(N_ENCOUNTERS > 500, 501, N_ENCOUNTERS),
    ENC_CATEGORY = if_else(N_ENCOUNTERS > 500, ">500", "≤500")
  )

# Histogram with distinct overflow bin annotation
p1 <- ggplot(hist_data, aes(x = N_ENC_BINNED, fill = PAYER_CATEGORY_PRIMARY)) +
  geom_histogram(binwidth = 20, color = "white") +
  geom_text(data = overflow_counts, aes(x = 510, y = Inf, label = glue(">{x_cap}: {n}")),
            vjust = 1.5, size = 3, hjust = 0) +
  facet_wrap(~ PAYER_CATEGORY_PRIMARY, scales = "free_y")
```

**Key insight:** Can't simply filter out >500 because that loses count information. Mutate to cap at 501, then annotate with actual overflow count per facet.

### Pattern 3: Filtering Masked Dates and Tracking Exclusions
**What:** Filter out placeholder dates (e.g., 1900) before plotting, but count excluded rows for footnote.

**When to use:** Bar charts by year where masked dates (1900, 9999, etc.) distort visualization (D-10, D-11).

**Example:**
```r
# Source: Project convention (D-10, D-11)
# Count masked dates before filtering
n_masked <- hl_cohort %>%
  filter(DX_YEAR == 1900) %>%
  nrow()

# Filter for plotting
enc_by_year <- hl_cohort %>%
  filter(!is.na(DX_YEAR), DX_YEAR != 1900) %>%
  group_by(DX_YEAR) %>%
  summarise(mean_post_tx_enc = mean(N_ENC_NONACUTE_CARE, na.rm = TRUE))

# Use n_masked in slide footnote
footnote_text <- glue("{n_masked} patients with masked diagnosis date (year 1900) excluded from this analysis")
```

**Key insight:** Filter for clean visualization, but track exclusions explicitly for transparency.

### Pattern 4: Preventing Bar Chart Label Clipping
**What:** Use `coord_cartesian(clip = "off")` combined with `ylim()` expansion to ensure `geom_text()` labels above bars don't get cut off at plot boundary.

**When to use:** Bar charts with `geom_text()` labels positioned above bars (vjust = -0.3) that clip at top (D-12).

**Example:**
```r
# Source: https://ggplot2.tidyverse.org/reference/coord_cartesian.html
# Expand y-axis to accommodate labels
max_y <- max(enc_by_year$mean_post_tx_enc, na.rm = TRUE)
p2 <- ggplot(enc_by_year, aes(x = DX_YEAR, y = mean_post_tx_enc)) +
  geom_col(fill = "#2c7fb8", alpha = 0.8) +
  geom_text(aes(label = round(mean_post_tx_enc, 1)), vjust = -0.3, size = 3) +
  coord_cartesian(clip = "off", ylim = c(0, max_y * 1.15)) +
  theme(plot.margin = margin(t = 10, r = 5, b = 5, l = 5))
```

**Key insight:** `clip = "off"` alone isn't enough — must also expand y-axis limits (multiply max by 1.1-1.15) to prevent labels from rendering outside panel.

### Pattern 5: Building Summary Statistics Table with dplyr
**What:** Use `group_by()` + `summarise()` with multiple summary functions (mean, median, quantiles) to create per-category stats.

**When to use:** Summary statistics slide showing per-payer encounter distribution (D-09).

**Example:**
```r
# Source: https://dplyr.tidyverse.org/reference/summarise.html
summary_stats <- hl_cohort %>%
  filter(!is.na(PAYER_CATEGORY_PRIMARY), !is.na(N_ENCOUNTERS)) %>%
  group_by(PAYER_CATEGORY_PRIMARY) %>%
  summarise(
    N = n(),
    Mean = round(mean(N_ENCOUNTERS, na.rm = TRUE), 1),
    Median = round(median(N_ENCOUNTERS, na.rm = TRUE), 1),
    Min = min(N_ENCOUNTERS, na.rm = TRUE),
    Q1 = round(quantile(N_ENCOUNTERS, 0.25, na.rm = TRUE), 1),
    Q3 = round(quantile(N_ENCOUNTERS, 0.75, na.rm = TRUE), 1),
    Max = max(N_ENCOUNTERS, na.rm = TRUE),
    `N > 500` = sum(N_ENCOUNTERS > 500, na.rm = TRUE),
    .groups = "drop"
  )
```

**Key insight:** Use `round()` for readability; backticks for column names with special characters.

### Pattern 6: Deleting Existing Slide
**What:** Use `remove_slide()` to delete a slide by index after reading existing PPTX.

**When to use:** Removing title slide (D-05) — though simpler to just not generate it.

**Example:**
```r
# Source: https://davidgohel.github.io/officer/reference/remove_slide.html
# NOT RECOMMENDED for this phase — simpler to comment out title slide generation
# But if needed:
pptx <- read_pptx("existing_deck.pptx")
pptx <- remove_slide(pptx, index = 1)  # Remove first slide
```

**Key insight:** For net-new generation scripts, commenting out slide code is cleaner than `remove_slide()`. Use `remove_slide()` only when editing existing PPTX files.

### Anti-Patterns to Avoid
- **Using `flextable::footnote()` for slide-level annotations:** `footnote()` adds superscript symbols to table cells with notes in table footer — doesn't place free text at slide bottom. Use `ph_with()` + `fpar()` instead.
- **Silently filtering outliers without tracking counts:** Histogram subtitle "X patients with >500 encounters not shown" is better than silent `coord_cartesian(xlim = c(0, 500))` which hides information.
- **Using `scale_x_continuous(limits = ...)` to exclude masked dates:** `limits` silently drops rows without count for footnote. Use `filter()` + `nrow()` explicitly.
- **Setting `clip = "off"` without expanding axis limits:** Labels still clip if y-axis max equals data max. Must expand by 10-15%.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| PowerPoint slide manipulation | Custom XML parsing of .pptx ZIP files | `officer::read_pptx()`, `add_slide()`, `ph_with()` | PPTX is complex XML in ZIP container; officer handles all relationships, layout refs, content types |
| Table styling in PPTX | Manual `ph_with()` calls for each cell | `flextable` + `style_table()` wrapper (exists in `11_generate_pptx.R:542`) | Flextable handles alternating rows, borders, alignment, font sizing consistently |
| Histogram overflow bin annotation | Hardcoded text positions via `annotate()` | Mutate to create overflow category + `geom_text()` with computed counts per facet | Faceting breaks hardcoded positions; computed per-facet counts scale automatically |
| Summary statistics by group | Manual column construction via `mutate()` | `group_by()` + `summarise()` with named list of functions | Reduces errors; named functions (mean = mean) create clean column names |

**Key insight:** Officer and flextable abstract OOXML complexity; direct XML manipulation risks breaking relationships. Histogram overflow bins need dynamic computation per facet (can't hardcode).

## Common Pitfalls

### Pitfall 1: Footnote Text Positioned Too High or Too Low
**What goes wrong:** Footnote text overlaps with table/image content or gets cut off at slide boundary.

**Why it happens:** Officer uses absolute positioning in inches. No "bottom anchor" — must calculate `top` parameter based on slide height.

**How to avoid:**
- Slide height is 5.625" (set in `11_generate_pptx.R:591`)
- Content typically ends at `top + height` = 1.1 + 4 = 5.1"
- Position footnote at `top = 5.0` to 5.2" (0.4" high) to fit within 5.625" boundary
- Test by opening PPTX and checking all slides

**Warning signs:** Footnote text invisible (off slide bottom) or overlapping table bottom row.

### Pitfall 2: Histogram Overflow Bin Shows Wrong Count per Facet
**What goes wrong:** Overflow bin annotation shows total count across all facets instead of per-facet count.

**Why it happens:** `geom_text()` with single data frame row repeats same count on all facets.

**How to avoid:**
```r
# WRONG: Single overflow count across all facets
n_beyond <- sum(hist_data$N_ENCOUNTERS > 500)
geom_text(aes(x = 510, y = Inf, label = glue(">{x_cap}: {n_beyond}")))

# RIGHT: Per-facet overflow counts
overflow_counts <- hist_data %>%
  filter(N_ENCOUNTERS > 500) %>%
  count(PAYER_CATEGORY_PRIMARY, name = "n_overflow")

geom_text(data = overflow_counts,
          aes(x = 510, y = Inf, label = glue(">{x_cap}: {n_overflow}")),
          vjust = 1.5, size = 3)
```

**Warning signs:** All facets show same overflow count despite visual differences in tail density.

### Pitfall 3: Bar Chart Labels Clip Despite `clip = "off"`
**What goes wrong:** Setting `coord_cartesian(clip = "off")` doesn't prevent label cutoff at top of plot.

**Why it happens:** `clip = "off"` allows drawing outside panel, but y-axis max still equals data max. Labels render at data_max + offset, which is outside plot area even with clipping disabled.

**How to avoid:**
```r
# Expand y-axis limits to 110-115% of data max
max_y <- max(data$value, na.rm = TRUE)
coord_cartesian(clip = "off", ylim = c(0, max_y * 1.15))
# Also add top margin for safety
theme(plot.margin = margin(t = 10, r = 5, b = 5, l = 5))
```

**Warning signs:** Labels disappear or partially clip at top edge; no error or warning messages.

### Pitfall 4: Filtering Masked Dates Loses Exclusion Count for Footnote
**What goes wrong:** `filter(DX_YEAR != 1900)` cleans data but footnote shows "X patients excluded" without calculating X first.

**Why it happens:** Filtering pipeline doesn't track intermediate row counts.

**How to avoid:**
```r
# Calculate exclusion count BEFORE filtering
n_masked <- hl_cohort %>%
  filter(DX_YEAR == 1900) %>%
  nrow()

# Then filter for plotting
enc_by_year <- hl_cohort %>%
  filter(!is.na(DX_YEAR), DX_YEAR != 1900) %>%
  group_by(DX_YEAR) %>%
  summarise(...)

# Use n_masked in footnote
footnote_text <- glue("{n_masked} patients with masked diagnosis date (year 1900) excluded")
```

**Warning signs:** Footnote says "N patients excluded" without actual number.

### Pitfall 5: Summary Stats Table Includes NA Payer Category as Row
**What goes wrong:** Summary statistics table shows "NA" as a payer category row alongside Medicare, Medicaid, etc.

**Why it happens:** `group_by(PAYER_CATEGORY_PRIMARY)` treats NA as a group level unless explicitly filtered.

**How to avoid:**
```r
# Filter out NA payers before grouping
summary_stats <- hl_cohort %>%
  filter(!is.na(PAYER_CATEGORY_PRIMARY), !is.na(N_ENCOUNTERS)) %>%
  group_by(PAYER_CATEGORY_PRIMARY) %>%
  summarise(...)
```

**Warning signs:** Summary table has 8 rows instead of 7 (6 payer categories + Missing).

### Pitfall 6: Deleting Title Slide Breaks Slide References in Code
**What goes wrong:** Commenting out or deleting title slide code causes "Slide 2" comments to become misleading (now Slide 1).

**Why it happens:** Slide numbers in code comments are hardcoded to original deck structure.

**How to avoid:**
- Option 1: Update all slide comments after deletion (Slide 2 → Slide 1, etc.)
- Option 2: Keep original slide numbering in comments with note: `# ---- Slide 2 (now Slide 1 after title removal): Insurance Coverage Overview ----`
- Option 3: Don't use slide numbers in comments — use descriptive names only

**Warning signs:** Code comment says "Slide 5" but it's actually Slide 4 in generated deck.
