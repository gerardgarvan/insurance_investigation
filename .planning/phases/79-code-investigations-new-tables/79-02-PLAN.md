**SECTION 3: PREPARE CANCER CODES**
The triggering_codes column in treatment_episodes.rds contains comma-separated ICD codes that triggered the treatment episode. These are the raw treatment procedure/drug codes, NOT cancer diagnosis codes.

For cancer codes, use the encounter-level cancer linkage approach:
- Episodes already have cancer_category from R/28 linkage
- Need raw ICD codes: extract from encounter-level linkage data if available, or derive from cancer_category

Key implementation (per D-15, D-16):
- Use triggering_codes from the episodes as the treatment code column
- For cancer codes: if treatment_episodes.rds contains a field with raw diagnosis ICD codes (check for columns like `cancer_dx_codes`, `linked_dx_codes`, or reconstruct from DuckDB DIAGNOSIS table using ENCOUNTERID)
- If no raw cancer ICD codes available in RDS, query DuckDB DIAGNOSIS table using ENCOUNTERID from episodes:
```r
# Get raw ICD codes for each encounter
USE_DUCKDB <- TRUE
open_pcornet_con()
dx_data <- get_pcornet_table("DIAGNOSIS") %>%
  filter(ENCOUNTERID %in% !!unique(na.omit(episodes$ENCOUNTERID))) %>%
  select(ENCOUNTERID, DX, DX_TYPE) %>%
  collect()

# Join cancer diagnosis codes to episodes
episode_dx <- episodes %>%
  left_join(
    dx_data %>%
      group_by(ENCOUNTERID) %>%
      summarise(cancer_codes = paste(sort(unique(DX)), collapse = ";"), .groups = "drop"),
    by = "ENCOUNTERID"
  )
```
- For episodes without ENCOUNTERID match, set cancer_codes to NA
- Use SEMICOLONS to separate multiple cancer codes (per D-15)
