**PART A: SCT Source Audit (per D-13, D-14, TREAT-01)**

Before modifying any extraction functions, add an SCT source audit section at the TOP of R/43a (after source/library calls, before extraction functions). This audit runs BEFORE the main extraction loop. The audit:

1. Calls the existing `extract_sct_dates()` function (which includes DX codes) to get patient set WITH DX codes.
2. Defines a temporary `extract_sct_dates_no_dx()` function inline that is identical to `extract_sct_dates()` but omits the DIAGNOSIS source section (section #2 in extract_sct_dates()). Calls it to get patient set WITHOUT DX codes.

   - `patients_with_dx <- n_distinct(sct_with_dx$ID)`
   - `patients_without_dx <- n_distinct(sct_without_dx$ID)`

   - `n_lost <- length(patients_lost)`
   - `retention_rate <- round(100 * patients_without_dx / patients_with_dx, 1)`
4. Stores the audit result in a tibble named `sct_audit_result` with columns: metric (character), value (character). Rows:
   - "Patients with SCT (WITH DX codes)", patients_with_dx
   - "Patients with SCT (WITHOUT DX codes)", patients_without_dx
   - "Patients lost (DX-only detection)", n_lost
   - "Retention rate", paste0(retention_rate, "%")
5. Logs the audit to console via message() with glue().
6. Saves `sct_audit_result` as `sct_audit_result.rds` in `CONFIG$cache$outputs_dir` for Plan 03 to include in the Phase 60 audit xlsx.

IMPORTANT: The audit must run BEFORE any code removal. The audit function `extract_sct_dates_no_dx()` is a temporary copy; the permanent removal happens next.
