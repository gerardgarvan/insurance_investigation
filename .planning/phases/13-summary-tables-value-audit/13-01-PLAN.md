**Header and sourcing:**
- Source R/01_load_pcornet.R (which auto-sources 00_config.R and utils)
- Load libraries: dplyr, purrr, stringr, glue, readr, tidyr
- Set output directory: file.path(CONFIG$output_dir, "tables", "value_audit")
- Create output directory with dir.create(recursive = TRUE, showWarnings = FALSE)

**Helper function 1: `suppress_hipaa(n_vec)`**
- Takes a numeric vector of counts
- Returns character vector where values 1-10 are replaced with "<11" (per D-04)
- 0 stays as "0", NA stays as NA, values >10 stay as formatted numbers with commas
- Percentages corresponding to suppressed counts should also be suppressed (replaced with "suppressed")

**Helper function 2: `audit_character_column(df, col_name)`**
- Counts distinct values: group_by value, summarise n = n()
- Calculates pct = n / sum(n) * 100, rounded to 2 decimal places
- Adds column_name as first column, column_type = "character"
- Sorts by descending n (most frequent first)
- Adds a row for NA values if any exist (value = "[NA]")
- Returns tibble with columns: column_name, column_type, value, n, pct, stat (stat = NA for character columns)

**Helper function 3: `audit_numeric_column(df, col_name)`**
- Computes: min, max, mean, median, n_missing, n_valid, n_distinct
- Returns tibble with same column structure but using stat column for stat names (min/max/mean/median/n_missing/n_valid/n_distinct)
- value column holds the stat value as character
- Apply HIPAA suppression: if n_valid is 1-10, suppress all stats

**Helper function 4: `audit_date_column(df, col_name)`**
- Computes: min date, max date, n_missing, n_valid
- Returns tibble with same structure, stat column for stat names
- Dates formatted as YYYY-MM-DD strings in value column
- Apply HIPAA suppression: if n_valid is 1-10, suppress all stats

**Helper function 5: `audit_table(df, table_name)`**
- Iterates over ALL columns in the dataframe (including wide TR tables with 314 columns -- per user requirement to include ALL columns)
- For each column, detects type:
  - is.character or is.factor -> audit_character_column
  - is.numeric or is.integer -> audit_numeric_column
  - inherits "Date" or "POSIXct" -> audit_date_column
  - is.logical -> treat as character (convert TRUE/FALSE/NA to strings, then audit_character_column)
- Binds all column results into one tibble
- Returns the combined tibble with table_name prepended as first column

**Main audit loop:**
```r
message(strrep("=", 60))
message("Value Audit: Enumerating all values across PCORnet CDM tables")
message(strrep("=", 60))

audit_results <- list()

for (tbl_name in names(pcornet)) {
  df <- pcornet[[tbl_name]]
  if (is.null(df)) {
    message(glue("  SKIP: {tbl_name} (NULL -- file not found during loading)"))
    next
  }

  message(glue("\nAuditing {tbl_name}: {format(nrow(df), big.mark=',')} rows, {ncol(df)} columns..."))

  result <- audit_table(df, tbl_name)
  audit_results[[tbl_name]] <- result

  # Apply HIPAA suppression to n and pct columns
  # For character audits: suppress n where 1-10, suppress pct where n is suppressed
  result <- result %>%
    mutate(
      n_raw = suppressWarnings(as.numeric(n)),
      n = case_when(
        is.na(n_raw) ~ n,  # Already non-numeric (like stat labels)
        n_raw >= 1 & n_raw <= 10 ~ "<11",
        TRUE ~ format(n_raw, big.mark = ",")
      ),
      pct = case_when(
        n == "<11" ~ "suppressed",
        TRUE ~ pct
      )
    ) %>%
    select(-n_raw)

  # Write CSV
  output_file <- file.path(output_dir, glue("{tbl_name}_values.csv"))
  write_csv(result, output_file)
  message(glue("  Wrote {format(nrow(result), big.mark=',')} rows to {output_file}"))
}
```

NOTE on HIPAA suppression logic: The suppress_hipaa helper can be used inside the audit functions, but the main loop should also do a final pass to ensure no counts 1-10 slip through. The `n` column in the final CSV should be character type (because it contains "<11" strings). The `pct` column should also be character (because it contains "suppressed" strings).

**Important implementation notes:**
- The audit functions should work on the pre-HIPAA-suppressed data (raw counts), then HIPAA suppression is applied as a final step before writing CSV. This is cleaner than suppressing inside each function.
- For the `n` and `pct` columns in the output CSV: store n as character (because "<11" is a string), store pct as character (because "suppressed" is a string). The raw numeric n should not appear in the final output.
- For character columns with very many distinct values (e.g., DX in DIAGNOSIS may have thousands of ICD codes), include ALL values. The point is exhaustive enumeration.
- For the _VALID logical columns added by the loader, convert to character ("TRUE"/"FALSE"/"NA") and audit as character. This shows the distribution of valid/invalid/missing per validation column.
- Do NOT skip any columns. The user explicitly wants ALL columns audited, especially for the wide TR tables.
