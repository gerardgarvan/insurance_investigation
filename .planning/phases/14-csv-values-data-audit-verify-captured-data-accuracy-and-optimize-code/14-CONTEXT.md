<decisions>
## Implementation Decisions

### Audit Scope
- **D-01:** Audit covers Phase 13 value_audit CSV output only (output/tables/value_audit/*.csv), not cohort or diagnostic pipeline CSVs
- **D-02:** User will run 17_value_audit.R on HiPerGator first, then bring the generated CSVs back for Claude to review directly in conversation (no automated validation script)

### Validation Approach
- **D-03:** Primary focus is coding inconsistencies — same concept coded differently across tables/columns (e.g., 'M'/'Male'/'MALE', mixed date formats, ICD code format mismatches) suggesting data capture problems
- **D-04:** This is a conversational review, not a scripted check — Claude reads the CSVs and discusses findings with the user

### Code Optimization
- **D-05:** Review scope is the full pipeline (all R scripts 01-17), not just 17_value_audit.R
- **D-06:** Style-preserving optimizations only — keep named predicate pattern and dplyr chains, but simplify redundant logic, remove dead code, reduce unnecessary data copies, improve vroom/readr usage
- **D-07:** No deep refactoring — don't restructure scripts, merge utilities, or change function signatures

### Findings Delivery
- **D-08:** Conversational review — Claude reads CSVs and discusses findings in chat, user decides what to act on, then Claude makes agreed-upon code changes
- **D-09:** No separate findings report or markdown output — findings are discussed live

### Claude's Discretion
- Claude determines which coding inconsistencies are worth flagging vs. expected PCORnet CDM variation
- Claude prioritizes optimization recommendations by impact (biggest speed/simplification gains first)

</decisions>
