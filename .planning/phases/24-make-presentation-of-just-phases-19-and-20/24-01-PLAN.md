  <action>
Create `R/22_generate_phase19_20_pptx.R` as a standalone script (not a modification of the full-deck script) using established officer/flextable/ggplot2 patterns.

Script structure:
1. **Setup and dependencies**
   - `library(officer)`, `library(flextable)`, `library(dplyr)`, `library(readr)`, `library(ggplot2)`, `library(scales)`, `library(glue)`, `library(stringr)`, `library(tidyr)`
   - define style constants matching existing deck (UF blue palette, fonts)
   - define helpers: `style_table()`, `add_table_slide()`, `add_image_slide()`, `add_footnote()`
2. **Read all Phase 19/20 CSV inputs**
   - Phase 19 UF: all five `uf_*` files
   - Phase 20 FLM: all four `flm_*` files
3. **Generate chart PNGs for the focused deck**
   - UF missingness by year (line chart or bar)
   - UF missingness by encounter type (bar)
   - FLM duplicate-rate/source comparison chart(s)
   - save under `output/figures/phase24_*`
4. **Build slides in this order**
   - title/overview slide for focused deck
   - Phase 19 section divider
   - Phase 19 tables/charts covering all selected outputs:
     - cross-summary
     - by year
     - by encounter type
     - year x encounter type (split across slides as needed)
     - raw vs harmonized
     - raw value distribution
   - Phase 20 section divider
   - Phase 20 tables/charts covering all selected outputs:
     - overall duplicate-rate summary
     - patient-level duplicate summary (split as needed)
     - date-level detail summarized (not raw full detail)
     - source payer completeness comparison
     - source-preference recommendation slide
   - concise closing/key-takeaways slide
5. **Large table handling**
   - split wide/tall tables across multiple slides (`D-06`)
   - ensure readable font sizes and subtitles indicating part number
6. **Save output**
   - `output_filename <- glue("insurance_tables_phase19_20_{Sys.Date()}.pptx")`
   - write PPTX in project root
   - print console summary with total slide count and output path

Critical constraints:
- Do not read any `all_source_*` or `all_site_*` files.
- Do not append to or overwrite the full deck flow from `R/11_generate_pptx.R`.
- Keep output deterministic and rerunnable from local/HiPerGator paths already used by project scripts.
  </action>
