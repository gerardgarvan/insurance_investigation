# Phase 84 Plan 01: Test Fixture Design & Creation Summary

**One-liner:** Created FIXTURE_DESIGN.md mapping 20 patients to 11 edge cases and generate_fixtures.R with tribble() definitions for all 15 PCORnet CDM tables.

## What Was Built

Two deliverables establishing the test fixture foundation:

1. **FIXTURE_DESIGN.md** - Human-readable test matrix documenting:
   - Patient roster: 20 patients (PT001-PT020) with edge case assignments
   - Data summary: Row counts and key features for all 15 tables
   - Edge case coverage matrix: Verification queries for each case
   - Technical notes: Payer codes, ICD codes, treatment codes, date ranges
   - Regeneration instructions and verification checklist

2. **generate_fixtures.R** - Executable R script that:
   - Sources R/00_config.R for PCORNET_TABLES, PCORNET_PATHS, and code constants
   - Defines 15 generate_*() functions using tribble() for readable data
   - Assembles fixture_tables list mapping table names to data
   - Writes all 15 CSVs via walk2() using PCORNET_PATHS for correct naming
   - Handles LAB_RESULT_CM filename override automatically
   - Uses na = "" in write_csv() to match production format

## Key Implementation Details

### Date Handling
All date columns are character strings (e.g., "2013-03-15") to match production CSV format and enable parse_pcornet_date() downstream. Dates centered on 2012-2014 to span ICD-9/ICD-10 transition.

### Code Constants Reused
- ABVD RXNORM_CUIs from R/00_config.R TREATMENT_CODES (lines 2389-2392)
- SCT CPT code 38241 from TREATMENT_CODES$sct_cpt
- Dual-eligible payer code "14" from AMC_PAYER_LOOKUP
- ICD-10 HL codes (C81.xx) from ICD_CODES$hl_icd10
- ICD-9 HL codes (201.xx) from ICD9_CANCER_SITE_MAP

### Column Schema Compliance
Every tribble() definition matches the corresponding {TABLE}_SPEC column order from R/01_load_pcornet.R. This ensures vroom can read fixtures without type mismatches.

### Empty Table Strategy
Tables not actively used by the pipeline (DISPENSING, MED_ADMIN, LAB_RESULT_CM, TUMOR_REGISTRY1/2/3) are empty tibbles with correct column names and types. This prevents "table not found" errors during DuckDB ingest while keeping fixture size minimal.

## Technical Decisions

1. **tribble() over data.frame()**: Row-oriented syntax is more readable for patient-centric test data
2. **walk2() for CSV writing**: Iterates over table names and data in parallel, using PCORNET_PATHS for consistent naming
3. **na = "" in write_csv()**: Ensures NA values become empty strings, matching production CSV format
4. **SOURCE column distributed**: All patients distributed across 5 OneFlorida+ sites (UFH, AMS, FLM, VRT, UMI) for realistic diversity
