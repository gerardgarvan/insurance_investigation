<objective>
Create R/111_doi_classification.R as a CLASSIFICATION-ONLY investigation script (D-05) that:
1. Pulls the DIAGNOSIS table from DuckDB using a native prefix pushdown (`WHERE LEFT(DX,3) IN (...)`) built from DOI_CODE_MAP keys — so only DoI-prefixed rows are ever materialized into R (D-01, DOI-CLASS-04 design constraint #4; OOM-safe on HiPerGator).
2. Refines the pushdown result in R with `is_doi_code()` (DX_TYPE-gated) + `classify_doi_codes()` to produce the encounter-level DoI classification across the FULL extract (all patients, all diagnosis positions P+S — D-01, D-03).
3. Runs the mutual-exclusivity hard-stop `sum(is_doi_code(DX) & is_cancer_code(DX)) == 0` and HALTS before writing any artifact if it fires (DOI-CLASS-04).
4. Attaches `paraneoplastic_flag = TRUE` to L10.81 encounters while keeping them in their pemphigus doi_category (D-04, DOI-CLASS-05).
5. Tags each encounter with `in_hl_cohort` from `get_hl_patient_ids()` (D-02).
6. Writes `doi_encounters.rds` at the (ID, ENCOUNTERID, DX_DATE, doi_code, doi_category) grain to `CONFIG$cache$outputs_dir`.

Purpose: doi_encounters.rds is the encounter-grain hand-off artifact that Plan 02 rolls up to patient grain and that Phase 129 (R/112) consumes for attribution. This plan owns everything up to and including the encounter-grain artifact.

Output: R/111_doi_classification.R (Sections 1–6: setup, prefix-list build, DuckDB pull, classify + paraneoplastic + in_hl_cohort, mutual-exclusivity hard-stop, doi_encounters.rds write). Plan 02 appends the patient-grain rollup section.
</objective>
