**Category 2: commented_code_linter (D-03) -- 57 violations across ~30+ files**

Search ALL R files in `R/*.R` and `R/utils/*.R` for patterns that look like commented-out code:
- Lines starting with `# ` followed by valid R expressions (assignments with `<-`, function calls like `filter(`, `mutate(`, `library(`, pipe operators `%>%`, etc.)
- Do NOT remove section headers (`# SECTION N: TITLE ----`), box-style headers (`# ======`), or explanatory WHY comments
- Do NOT remove `# nolint` annotations

Decision rule per D-03 and 71-RESEARCH.md:
- If the commented line is orphaned old code with no explanatory context: DELETE it
- If the commented line documents a past decision (e.g., "we tried X"): CONVERT to a WHY comment that explains the decision without showing code (e.g., `# NOTE: Age filter removed in Phase 43 to include pediatric patients`)
- If unclear whether it's documentation or dead code: DELETE it (git history preserves the original per D-03)

**Category 3: indentation_linter (D-05) -- 27 violations**

Search all R files for inconsistent indentation:
- All code inside functions, if/else blocks, for loops, and pipe chains must use 2-space indentation
- Pipe continuations must be indented 2 spaces from the pipe start
- No tabs (use spaces only)
- Closing parentheses/braces should align with the opening statement

**Category 4: pipe_continuation_linter (D-05) -- 30 violations**

Search all R files for pipes that don't follow tidyverse continuation style:
- Each `%>%` must appear at the END of a line, never at the start
- The function after `%>%` starts on the next line, indented 2 spaces

Fix pattern:
```r
# BEFORE (violation)
result <- data %>% filter(condition) %>%
  select(columns)

# AFTER (correct)
result <- data %>%
  filter(condition) %>%
  select(columns)
```

- Function calls with 5+ arguments: one argument per line
- Do NOT wrap complex dplyr pipelines that are 150-160 chars if wrapping hurts readability
- Target: bring total remaining violations under 50 (not zero)
