CRITICAL implementation notes:
- Per D-03: Aggregate counts only. NEVER include patient IDs (ID column) in output dataframes. Select only code, code_type, source_table columns before aggregating.
- Per D-04: Query ALL patients in raw data. Do NOT call any cohort pipeline functions (no has_hodgkin_diagnosis, no build_cohort, etc.).

- Per D-07: Raw code values only, no description lookup.

- Per pitfall 3: Wrap every table access in safe_table() + null guard.
- Per pitfall 4: All hex colors use 8-char format with "FF" alpha prefix.
- For DuckDB lazy queries: push group_by/summarise to SQL layer, collect() only the summary. For ICD-10-PCS prefix matching: filter PX_TYPE first (lazy), then materialize() for R-side str_detect().
