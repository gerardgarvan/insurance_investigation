From Python reference (PAYER_VARIABLES_AND_CATEGORIES.md):
```
9 categories: Medicare, Medicaid, Dual eligible, Private, Other government,
              No payment / Self-pay, Other, Unavailable, Unknown

Effective payer: primary if valid, else secondary if valid, else null
Sentinels: null, empty, NI, UN, OT (trigger fallback)
99/9999: NOT sentinel -- map to "Unavailable"

Dual-eligible (encounter-level):
  (a) Medicare primary (prefix 1) + Medicaid secondary (prefix 2), OR
  (b) Medicaid primary + Medicare secondary, OR
  (c) Primary OR secondary in {14, 141, 142}
  When secondary missing: dual_eligible = 0

Category mapping order (case_when top-to-bottom):
  1. Exact-match overrides: 99/9999 -> Unavailable; NI/UN/OT/UNKNOWN/NA -> Unknown
  2. Prefix rules: 1->Medicare, 2->Medicaid, 5/6->Private, 3/4->Other gov, 8->Self-pay, 7/9->Other
  3. Dual-eligible override: if encounter dual_eligible=1, category becomes "Dual eligible"
  4. Default: "Other"
```

**SECTION 1 -- Named payer functions (per D-13):**

1. `compute_effective_payer(primary, secondary)` -- Per D-02:
   - Define `sentinel_values <- PAYER_MAPPING$sentinel_values` (i.e., c("NI", "UN", "OT"))
   - `primary_valid <- !is.na(primary) & nchar(trimws(primary)) > 0 & !primary %in% sentinel_values`
   - `secondary_valid <- !is.na(secondary) & nchar(trimws(secondary)) > 0 & !secondary %in% sentinel_values`
   - Return: `case_when(primary_valid ~ primary, secondary_valid ~ secondary, TRUE ~ NA_character_)`
   - Note: 99/9999 are NOT sentinel per D-03; they pass validity check and map to "Unavailable" in category step

2. `detect_dual_eligible(primary, secondary)` -- Per D-01, D-04:
   - Define `dual_codes <- PAYER_MAPPING$dual_eligible_codes` (i.e., c("14", "141", "142"))
   - When secondary is missing/empty: return 0L (per D-04)
   - Check: `secondary_missing <- is.na(secondary) | nchar(trimws(secondary)) == 0`
   - Check dual codes: `has_dual_code <- primary %in% dual_codes | secondary %in% dual_codes`
   - Check cross-payer: `(str_starts(primary, "1") & str_starts(secondary, "2")) | (str_starts(primary, "2") & str_starts(secondary, "1"))`
   - Return: `case_when(secondary_missing ~ 0L, has_dual_code ~ 1L, cross_payer ~ 1L, TRUE ~ 0L)`
   - Use `stringr::str_starts()` (NOT str_starts_with -- that doesn't exist in stringr; use `str_starts(x, "1")`)

3. `map_payer_category(effective_payer, dual_eligible_encounter)` -- Per D-05, D-03:
   - First compute raw category from effective_payer using case_when() with EXACT-MATCH OVERRIDES BEFORE PREFIX RULES:
     ```
     payer_category_raw = case_when(
       effective_payer %in% PAYER_MAPPING$unavailable_codes ~ "Unavailable",          # 99, 9999
       effective_payer %in% PAYER_MAPPING$unknown_codes | is.na(effective_payer) ~ "Unknown",  # NI, UN, OT, UNKNOWN, NA
       str_starts(effective_payer, "1") ~ "Medicare",
       str_starts(effective_payer, "2") ~ "Medicaid",
       str_starts(effective_payer, "5") | str_starts(effective_payer, "6") ~ "Private",
       str_starts(effective_payer, "3") | str_starts(effective_payer, "4") ~ "Other government",
       str_starts(effective_payer, "8") ~ "No payment / Self-pay",
       str_starts(effective_payer, "7") | str_starts(effective_payer, "9") ~ "Other",
       TRUE ~ "Other"
     )
     ```
   - Then apply dual-eligible override per D-05: `if_else(dual_eligible_encounter == 1L, "Dual eligible", payer_category_raw)`
