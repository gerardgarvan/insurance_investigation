Create R/21_all_site_duplicate_dates.R -- a standalone diagnostic script that extends Phase 20's FLM-specific duplicate date investigation (R/19_flm_duplicate_dates.R) to ALL 5 partner sites (AMS, UMI, FLM, VRT, UFH). Purpose: Enable the user to assess duplicate date data quality across ALL OneFlorida+ sites, not just FLM, and identify per-site source preferences for payer data when multi-source duplicates exist.

Output: R/21_all_site_duplicate_dates.R (standalone script), 5 CSVs in output/tables/ with all_site_ prefix

**SECTION 2: Build all-site encounter dataset (D-02, D-03)**
Per D-02: Examine ENCOUNTER.SOURCE within each patient's encounters.
CRITICAL: Handle the two SOURCE column collision (ENCOUNTER.SOURCE vs DEMOGRAPHIC.SOURCE). Use the pattern from Phase 21 SUMMARY (select(-SOURCE) or rename). Specifically:
```r
all_encounters <- pcornet$ENCOUNTER %>%
  rename(ENCOUNTER_SOURCE = SOURCE) %>%
  left_join(pcornet$DEMOGRAPHIC %>% select(ID, SOURCE), by = "ID") %>%
  rename(SITE = SOURCE)
```
Now SITE = patient's home site (from DEMOGRAPHIC), ENCOUNTER_SOURCE = where encounter was recorded.
- Log N encounters per SITE
- Log NA ENCOUNTER_SOURCE counts per SITE
- Log unique ENCOUNTER_SOURCE values per SITE
- Warn if any SITE is NA (encounters with no DEMOGRAPHIC record -- use `mutate(SITE = if_else(is.na(SITE), "<No Site>", SITE))`)

**SECTION 3: Same-date duplicate detection (D-03, ALLDUP-01)**
Per D-03: Same duplicate definitions as Phase 20 -- group by ID + date only (not ENC_TYPE), check ADMIT_DATE primary + DISCHARGE_DATE secondary.
- Parse ADMIT_DATE and DISCHARGE_DATE with `as.Date(ADMIT_DATE, format = "%Y-%m-%d")`
- If standard parse rate < 50%, fall back to parse_pcornet_date() (same as Phase 20)
- Log parse success rate
- ADMIT_DATE same-date duplicates: `add_count(SITE, ID, admit_date_parsed, name = "n_encounters_same_date")` then filter n > 1
- Log per-SITE duplicate counts: n_dupe_rows, n_dupe_patient_dates, n_dupe_patients
- DISCHARGE_DATE same-date duplicates: same approach with SITE grouping, log per-SITE counts

**SECTION 4: Exact row duplicates (D-03, ALLDUP-01)**
- Exact row duplicates: `get_dupes()` on all original columns (excluding parsed date columns and SITE)
- Near-exact duplicates: `get_dupes()` excluding ENCOUNTERID as well
- Log counts per SITE by joining back to SITE assignment
- Handle sites with zero exact/near-exact duplicates gracefully

**SECTION 5: Multi-source date identification (D-02, ALLDUP-02)**
Per D-02: Examine ENCOUNTER_SOURCE variation within each patient's encounters.
- From admit_date_dupes, group by SITE, ID, admit_date_parsed
- Compute n_sources = n_distinct(ENCOUNTER_SOURCE, na.rm = TRUE), sources = paste(sort(unique(na.omit(ENCOUNTER_SOURCE))), collapse = ", "), enc_types, n_enc_types
- Filter to multi_source_dates where n_sources > 1
- Breakdown per SITE: same-source vs multi-source duplicates
- Within same-source per SITE: same ENC_TYPE (potential true duplication) vs different ENC_TYPE (clinically valid)
- Log most common source combinations per SITE (top 5 per site)

**SECTION 6: Payer completeness comparison (D-08, D-09, ALLDUP-03)**
Per D-08: Per-site source-preference recommendations.
Per D-09: Each site gets its own recommendation based on its own payer completeness rates.
- Get multi-source encounter rows via semi_join on multi_source_dates
- If no multi-source encounters for a site, log "No multi-source dates found for {SITE}" and create empty source_completeness for that site
- Per-SITE, per-ENCOUNTER_SOURCE completeness: n_encounters, n_primary_present, pct_primary_present, n_secondary_present, pct_secondary_present, n_both_present, pct_both_present, n_either_present, pct_either_present
- Generate per-site recommendation: for each SITE, the ENCOUNTER_SOURCE with highest pct_primary_present
- Log recommendations per site

**SECTION 7: Build and write CSV outputs (D-05, D-06, D-07, ALLDUP-04, ALLDUP-05)**
Per D-05: 5 CSV files with SITE column + cross-site summary.
Per D-06: all_site_ prefix. Per D-07: output to output/tables/.

Same structure as Phase 20's flm_patient_duplicate_summary.csv but with SITE column added.
Columns: SITE, ID, n_unique_dates, n_total_encounters, n_duplicate_dates, n_multi_source_dates, sources_in_encounters, pct_primary_present, pct_secondary_present.
Sorted by SITE, then desc(n_multi_source_dates), desc(n_duplicate_dates).

CSV 2: `all_site_date_level_duplicate_detail.csv` -- date-level detail for multi-source encounters with SITE column.
Columns: SITE, ID, admit_date_parsed, ENCOUNTER_SOURCE, ENC_TYPE, ENCOUNTERID, ADMIT_TIME, DISCHARGE_DATE, DISCHARGE_TIME, PAYER_TYPE_PRIMARY, PAYER_TYPE_SECONDARY, primary_missing, secondary_missing.
If no multi-source encounters exist for any site, write empty tibble with expected columns.

CSV 3: `all_site_duplicate_aggregate_summary.csv` -- per-SITE aggregate metrics.
One block of rows per SITE with metrics: Total patients (DEMOGRAPHIC), Total encounters, Encounters with valid ADMIT_DATE, Encounters with NA ADMIT_DATE, Encounters with NA ENCOUNTER_SOURCE, Unique patient-dates, Patient-dates with same-date duplicates, Patient-dates with multiple SOURCEs, Exact row duplicates, Near-exact duplicates, DISCHARGE_DATE same-date duplicate patient-dates.
Columns: SITE, metric, value. Append source completeness rows if available per site.

CSV 4: `all_site_source_payer_completeness.csv` -- per-SITE per-ENCOUNTER_SOURCE completeness.
Columns: SITE, ENCOUNTER_SOURCE, n_encounters, n_primary_present, pct_primary_present, n_secondary_present, pct_secondary_present, n_both_present, pct_both_present, n_either_present, pct_either_present.
