Create R/utils_dates.R containing the parse_pcornet_date() function (per D-20).

The function accepts a character vector and returns a Date vector. It implements a fallback chain (per D-09):

1. **Attempt 1: YYYY-MM-DD (ISO format)** -- Use lubridate::ymd() with quiet=TRUE. Most common in recent SAS exports.

2. **Attempt 2: Excel serial numbers** -- For remaining NAs, try as.numeric() and pass to janitor::excel_numeric_to_date(). Only attempt if value is numeric string and in valid range (1 < x < 100000). This handles Excel-origin dates.

3. **Attempt 3: SAS DATE9 (DDMMMYYYY like 15JAN2020)** -- For remaining NAs, use lubridate::parse_date_time() with orders = "dby", quiet = TRUE. Then convert to Date.

4. **Attempt 4: YYYYMMDD (compact, no separators)** -- For remaining NAs where string length is exactly 8, try lubridate::ymd() again (handles 20200115 format).

5. **Log unparseable count** -- After all attempts, count NAs that were not NA in input. Log with message() and glue(): `"  WARNING: {unparsed_count} ({unparsed_pct}%) dates could not be parsed"`.

6. **Return Date vector** -- Return the result with NAs for truly unparseable values.

File structure:

Create R/utils_attrition.R containing two functions (per D-16):

**init_attrition_log()** -- Creates an empty data frame with columns (per D-18):
- step: character (filter step name)
- n_before: integer (unique patient count before this step)
- n_after: integer (unique patient count after this step)
- n_excluded: integer (n_before - n_after)
- pct_excluded: numeric (percentage excluded, rounded to 1 decimal)

Use `stringsAsFactors = FALSE` for R < 4.1 compatibility.

**log_attrition(log_df, step_name, n_after)** -- Appends a new row:
- If log_df has 0 rows (first step), set n_before = n_after (no exclusions for initial population)
- Otherwise, n_before = last row's n_after
- Calculate n_excluded = n_before - n_after
- Calculate pct_excluded = if(n_before > 0) round(100 * n_excluded / n_before, 1) else 0
- Print a message with glue: `"[Attrition] {step_name}: {n_before} -> {n_after} ({n_excluded} excluded, {pct_excluded}%)"` (per D-22: simple message() calls)
- Return rbind(log_df, new_row) -- the updated log
