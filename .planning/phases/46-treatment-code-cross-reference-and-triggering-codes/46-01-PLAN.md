Create R/46_treatment_cross_reference.R with these sections:

**SECTION 1: SETUP** — Load libraries (dplyr, stringr, glue, purrr, openxlsx2), source R/00_config.R and R/01_load_pcornet.R. Define output path: file.path(CONFIG$output_dir, "tables", "treatment_cross_reference.xlsx").

**SECTION 2: HARDCODED REFERENCE DATA** — Define REFERENCE_CODES as a named list mirroring TREATMENT_CODES structure, with sub-lists per treatment type (chemotherapy, radiation, sct, immunotherapy). Each sub-list contains named character vectors of codes from the merged docx/xlsx sources. Use the schemas documented in Task 1.

Per D-02: Merge both TreatmentVariables_2024.07.17.docx (broad, 6 categories) AND Treatment_Variable_Documentation.docx (adds Q-codes, ICD-10-PCS patterns, doxorubicin detail).
Per D-03/D-16: Include codes from PCS Codes Cancer Tx.xlsx, ComprehensiveSurgeryCodes.xlsx, MSDRGs.xlsx as hardcoded data.

Also hardcode PHASE45_ADDED_CODES vector (the 42 radiation CPT codes from Phase 45 auto-add).

**SECTION 3: COMPARISON FUNCTIONS** — Define compare_code_lists(reference, config) returning list(in_ref_not_config, in_config_not_ref) using setdiff(). For prefix-based codes (ICD-10-PCS), compare prefixes directly. For CPT ranges (radiation 70010-79999), generate range-level narrative per D-04/D-05 instead of code-level expansion.

**SECTION 4: RUN COMPARISONS** — For each treatment type, run comparisons across all code categories. Collect results into a list of tibbles per type. Each row: code, direction ("In Reference Not Config" or "In Config Not Reference"), code_category (e.g., "HCPCS J-codes", "ICD-9", "DRG"), source_document, annotation (Phase 45 annotation per D-13 where applicable).

For RxNorm CUIs in config: add a note row "RxNorm CUIs: N codes in config; not specified in reference documents (drug lookup-derived)" rather than treating as gaps.

**SECTION 5: DUCKDB PATIENT/ENCOUNTER COUNTS** — For each gap code (in_ref_not_config), query PROCEDURES using get_pcornet_table("PROCEDURES") to get patient_count and encounter_count per D-14. Use the Phase 45 pattern: group_by(PX) %>% summarise(patient_count = n_distinct(ID), encounter_count = n()). Left-join counts onto the gap tibbles.

**SECTION 6: STYLED XLSX OUTPUT** — Create multi-sheet workbook per D-11:
- Sheet 1 "Summary": Overall counts per type, total in-ref-not-config, total in-config-not-ref, plus range-level narratives for radiation CPT per D-04/D-05.
- Sheets 2-5: One per treatment type (Chemotherapy, Radiation, SCT, Immunotherapy) per D-11, D-12. Each row shows: code, direction, code_category, source_document, patient_count, encounter_count, annotation.
- Use TREATMENT_TYPE_COLORS for per-type header styling.
- Use int2col() (NOT int_to_col()), format(x, big.mark=',') for thousands.
- Per D-13: codes in PHASE45_ADDED_CODES get annotation "Added via Phase 45 audit -- confirmed treatment codes in patient data".

**SECTION 7: CONSOLE SUMMARY** — Log total gaps per type, highlight any reference codes with patient data (potential config additions).

Key anti-patterns to avoid:
- Do NOT use int_to_col() — use int2col()
- Do NOT use glue("{x:,}") — use format(x, big.mark=',')
- Do NOT expand ranges to individual codes for radiation CPT — use range-level narrative per D-04
- Do NOT parse docx/xlsx at runtime — all reference data is hardcoded per D-15/D-16
