**Step 1: Create extract_dates_with_codes(type) in R/44_treatment_episodes.R**

Add a new function `extract_dates_with_codes(type)` that mirrors the logic of `extract_all_dates(type)` from R/43_treatment_durations.R but returns 3 columns: `ID`, `treatment_date`, and `triggering_code`. The key difference from extract_all_dates() is that `select()` calls include `triggering_code = PX` (for PROCEDURES sources) or an appropriate code column for other sources.

For each treatment type, replicate the source-specific extraction logic from R/43_treatment_durations.R but add `triggering_code`:

- **PROCEDURES source**: `select(ID, treatment_date = PX_DATE, triggering_code = PX)` — this is the primary source of meaningful bare codes (CPT, HCPCS, ICD-9, ICD-10-PCS, revenue codes).
- **PRESCRIBING/DISPENSING/MED_ADMIN sources** (chemo only): `select(ID, treatment_date = ..., triggering_code = RXNORM_CUI)` — RxNorm CUIs are valid bare codes per D-08.
- **DIAGNOSIS source**: `select(ID, treatment_date = DX_DATE, triggering_code = DX)` — Z51.0, Z51.11, V58.0 etc. are bare codes that read as ICD format per D-08.
- **ENCOUNTER/DRG source**: `select(ID, treatment_date = ADMIT_DATE, triggering_code = DRG)` — DRG codes like "849", "018" are bare numeric codes.
- **TUMOR_REGISTRY source**: `select(ID, treatment_date = ..., triggering_code = NA_character_)` — no individual code; TR dates are date evidence only.

Create a `stack_and_dedup_with_codes()` helper that stacks sources and deduplicates on `distinct(ID, treatment_date, triggering_code)` (3-column distinct, not 2-column). This preserves multiple codes on the same date while removing exact duplicates.

4. The final `select()` adds `triggering_codes` as the LAST column (after historical_flag) per Pitfall 5
