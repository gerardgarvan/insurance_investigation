<objective>
Create a standalone diagnostic R script that investigates payer data missingness for UFH (University of Florida) patients in the OneFlorida+ HL cohort, profiling both raw ENCOUNTER PAYER_TYPE fields and derived harmonized categories, with breakdowns by year, encounter type, and their combination.

Purpose: Characterize why insurance/payer data is missing for UF patients. The primary hypothesis is a data submission gap -- certain encounter types or time periods may systematically lack payer information from UF.

Output: `R/18_uf_insurance_missingness.R` script + 5 CSV files in `output/tables/` with missingness breakdowns.
</objective>

  <action>
Create `R/18_uf_insurance_missingness.R` as a standalone diagnostic script with these exact sections:

**HEADER:** Standard comment block identifying this as Phase 19, standalone (not part of main pipeline sequence), listing requirements UFMISS-01 through UFMISS-04, with usage instructions to `source("R/18_uf_insurance_missingness.R")`.

**DEPENDENCIES:**
```r
source("R/02_harmonize_payer.R")  # Loads 01 -> 00 chain; provides encounters tibble with payer_category
library(dplyr)
library(lubridate)
library(glue)
library(readr)
library(stringr)
library(tidyr)
```
Note: sourcing 02_harmonize_payer.R automatically sources 01_load_pcornet.R which sources 00_config.R. This gives us both raw `pcornet$ENCOUNTER`, `pcornet$DEMOGRAPHIC`, the `encounters` tibble (with `payer_category`), and `PAYER_MAPPING`.

**SECTION 1: Identify UFH patients.**
- Filter `pcornet$DEMOGRAPHIC` to `SOURCE == "UFH"`, select `ID, SOURCE`, `distinct()`.
- Log count: `message(glue("UFH patients in dataset: {format(nrow(ufh_patients), big.mark=',')}"))`

**SECTION 2: Build UFH encounter dataset with missingness flags.**
- Inner join `pcornet$ENCOUNTER` to `ufh_patients` by `ID`.
- Define `missing_indicators <- c(PAYER_MAPPING$sentinel_values, PAYER_MAPPING$unavailable_codes)` -- this is `c("NI", "UN", "OT", "99", "9999")` per D-01 to D-04.
- Add missingness columns per D-02:
  - `primary_missing`: `is.na(PAYER_TYPE_PRIMARY) | nchar(trimws(PAYER_TYPE_PRIMARY)) == 0 | PAYER_TYPE_PRIMARY %in% missing_indicators`
  - `secondary_missing`: same logic for `PAYER_TYPE_SECONDARY`
  - `both_missing`: `primary_missing & secondary_missing`
- Log total UFH encounters and overall primary/secondary/both missingness rates.

**SECTION 3: Raw PAYER_TYPE_PRIMARY value distribution (UFMISS-01).**
- `count(ufh_encounters, PAYER_TYPE_PRIMARY, sort = TRUE)` then `mutate(pct = round(100 * n / sum(n), 1))`.
- Also do the same for `PAYER_TYPE_SECONDARY`.
- Combine into a single CSV with a `field` column ("PRIMARY" / "SECONDARY"), columns: `field`, `value`, `n`, `pct`.
- Replace NA values in the value column with the string `"<NA>"` for CSV readability.
- Write to `output/tables/uf_payer_raw_value_distribution.csv`.
- Log top 5 PRIMARY values to console.

**SECTION 4: Temporal breakdown by year (UFMISS-02, D-07).**
- Log and exclude encounters with `is.na(ADMIT_DATE)`: `n_no_admit_date <- sum(is.na(ufh_encounters$ADMIT_DATE))`, log count.
- Log and exclude 1900 sentinel dates: `n_sentinel_dates <- sum(year(ufh_encounters$ADMIT_DATE) == 1900L, na.rm = TRUE)`, log count.
- Create `ufh_encounters_valid` filtering out both NA dates and 1900 sentinels.
- Extract `admit_year = year(ADMIT_DATE)`.
- Group by `admit_year`, summarise: `n_encounters`, `n_primary_missing = sum(primary_missing)`, `n_secondary_missing = sum(secondary_missing)`, `n_both_missing = sum(both_missing)`, `pct_primary_missing = round(100 * n_primary_missing / n_encounters, 1)`, `pct_secondary_missing`, `pct_both_missing`.
- Arrange by `admit_year`.
- Write to `output/tables/uf_payer_missingness_by_year.csv`.
- Log each year's primary missingness to console in a for loop.

**SECTION 5: Encounter type breakdown (UFMISS-02, D-07).**
- Preserve NA ENC_TYPE as `"<NA>"` for visibility (per Pitfall 3 from RESEARCH.md): `mutate(ENC_TYPE_LABEL = if_else(is.na(ENC_TYPE), "<NA>", ENC_TYPE))`.
- Group by `ENC_TYPE_LABEL`, summarise same columns as Section 4.
- Arrange by `desc(n_encounters)`.
- Write to `output/tables/uf_payer_missingness_by_enc_type.csv`.
- Log each encounter type's missingness to console.

**SECTION 6: Year x Encounter Type crosstab (UFMISS-02, D-07).**
- Use `ufh_encounters_valid` (already has valid dates).
- Apply same `ENC_TYPE_LABEL` transformation.
- Group by `admit_year, ENC_TYPE_LABEL`, summarise: `n_encounters`, `n_primary_missing`, `pct_primary_missing`.
- Arrange by `admit_year, ENC_TYPE_LABEL`.
- Write to `output/tables/uf_payer_missingness_year_x_enc_type.csv`.
- Log the top 10 year x type combinations with highest missingness pct (where n_encounters >= 10).

**SECTION 7: Raw vs Harmonized comparison (UFMISS-03, D-08).**
- Get encounter-level harmonized categories from the `encounters` tibble (produced by 02_harmonize_payer.R sourced in dependencies). Inner join to `ufh_patients` by `ID`.
- Filter out NA ADMIT_DATE and 1900 sentinels (same as Section 4).
- Compute `harmonized_missing = is.na(payer_category) | payer_category %in% c("Unknown", "Unavailable")`.
- Compute raw `primary_missing` using same logic as Section 2.
- Overall comparison: `n_encounters`, `n_raw_primary_missing`, `pct_raw_primary`, `n_harmonized_missing`, `pct_harmonized`. This should be a single summary row.
- Also compute per-year comparison: group by `year(ADMIT_DATE)`, both raw and harmonized missingness.
- Combine overall + per-year into one data frame (overall row has `year = "OVERALL"`).
- Write to `output/tables/uf_payer_raw_vs_harmonized.csv`.
- Log the overall comparison to console with interpretation: if harmonized pct differs from raw, note whether harmonization is increasing or decreasing apparent missingness.

**SECTION 8: Console summary (D-14, exploratory).**
- Print a summary block with:
  - Total UFH patients, total UFH encounters
  - Excluded: N encounters with no ADMIT_DATE, N encounters with 1900 sentinel
  - Overall primary/secondary/both missingness rates
  - Year range with highest primary missingness (>50% if exists)
  - Encounter type with highest primary missingness rate (minimum 10 encounters)
  - Raw vs harmonized delta
  - List of CSV files written

**CSV OUTPUT:** Use `readr::write_csv()` to `file.path(CONFIG$output_dir, "tables", filename)`. Ensure `dir.create(output_dir, showWarnings = FALSE, recursive = TRUE)` at start of output section.
