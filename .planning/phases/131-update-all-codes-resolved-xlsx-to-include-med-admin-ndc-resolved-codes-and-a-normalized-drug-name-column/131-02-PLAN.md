<objective>
Generalize R/50's MED_ADMIN/DISPENSING NDC-resolved code detection (the Phase 122 fix) across all 4 RXNORM-based treatment vector categories, and make the "Source Table" column reflect which table/type actually matched each code instead of a fixed per-vector label.

Purpose: R/50 today only queries PRESCRIBING and MED_ADMIN with `MEDADMIN_TYPE == "RX"` — it has never queried DISPENSING at all, and it has never resolved `MEDADMIN_TYPE == "ND"` (NDC-typed) rows via the Phase 122 crosswalk. This means codes only reachable through those paths are invisible in `all_codes_resolved.xlsx` today, even though 5 other consumers (R/10, R/11, R/25, R/26, R/76) were already fixed in Phase 122. This plan closes that "one remaining consumer" gap explicitly deferred at the end of Phase 122, and does so generically (not chemo-only) since R/50's RXNORM loop already iterates chemo_rxnorm, sct_rxnorm, immunotherapy_rxnorm, and supportive_care_rxnorm uniformly.

Output: `get_chemo_hits()` gains an additive, backward-compatible source-tagging parameter; R/50's Section 3 RXNORM aggregation is rewritten to use it across 3 tables x 4 vectors with correct de-duplication; Section 4's `all_codes_df` assembly carries the real per-code source label through instead of `code_type_map`'s static string.
</objective>
