<objective>
Create a standalone gap analysis script (R/09_dx_gap_analysis.R) that investigates the 19 patients excluded as "Neither" (no HL evidence in DIAGNOSIS or TUMOR_REGISTRY) to characterize the data gap by pulling their full diagnosis history, filtering to lymphoma/cancer codes, cross-referencing ENROLLMENT and TUMOR_REGISTRY tables, classifying each patient's gap type by site, and producing focused CSV outputs plus a console summary.

Purpose: Determine whether the 19 excluded patients represent missed HL codes (closable gap) or a fundamental data quality limitation (non-HL patients in the extract), enabling an informed decision about whether to expand HL identification criteria or document the gap as-is.

Output: R/09_dx_gap_analysis.R script, three diagnostic CSVs (neither_all_diagnoses.csv, neither_lymphoma_codes.csv, neither_patient_summary.csv), and console summary with site-stratified gap classification.
</objective>

**Section 1: Load excluded patients** (per D-10)
- Read `output/cohort/excluded_no_hl_evidence.csv` with `read_csv(..., show_col_types = FALSE)`
- Store in `excluded_patients` tibble
- Log count and sites: `message(glue("Loaded {nrow(excluded_patients)} excluded patients"))`
- Log per-site breakdown: `message(glue("Sites: {paste(sort(unique(excluded_patients$SOURCE)), collapse = ', ')}"))`
- If file does not exist or has 0 rows, stop with informative message: `stop("No excluded patients found. Run the full pipeline first (source R/04_build_cohort.R).")`

**Section 2: DIAGNOSIS table exploration** (per D-01)
- Use `semi_join(excluded_patients, by = "ID")` to filter pcornet$DIAGNOSIS to only the 19 patients' records
- Select columns: ID, DX, DX_TYPE, DX_DATE, DX_SOURCE, ADMIT_DATE, SOURCE (include SOURCE from DIAGNOSIS table for site attribution)
- Store in `neither_dx`
- Log: `message(glue("Found {nrow(neither_dx)} diagnosis records for {n_distinct(neither_dx$ID)} Neither patients"))`
- Also log patients with ZERO diagnosis records: compute `no_dx_patients` as `anti_join(excluded_patients, neither_dx, by = "ID")` and log count

**Lymphoma/cancer code filtering** (per D-01 focused summary):
- Normalize DX codes using `normalize_icd()` from utils_icd.R
- Define ICD-10 lymphoma pattern: `lymphoma_icd10_pattern <- "^C(8[1-9]|9[0-6])"` (matches C81-C96)
- Define ICD-9 lymphoma pattern: `lymphoma_icd9_pattern <- "^20[0-8]"` (matches 200-208)
- Filter `neither_dx` to rows matching either pattern (based on DX_TYPE "10" or "09")
- Store in `neither_lymphoma`
- Log: `message(glue("Found {nrow(neither_lymphoma)} lymphoma/cancer codes for {n_distinct(neither_lymphoma$ID)} patients"))`
- Also check if any of these are actual HL codes using `is_hl_diagnosis()` and log if found (this would indicate a bug in the pipeline)

**Section 3: ENROLLMENT cross-reference** (per D-02, D-05)
- Use `semi_join(excluded_patients, by = "ID")` on pcornet$ENROLLMENT
- Group by ID, SOURCE and summarise: `has_enrollment = TRUE`, `enr_start = min(ENR_START_DATE, na.rm = TRUE)`, `enr_end = max(ENR_END_DATE, na.rm = TRUE)`, `n_enrollment_records = n()`
- Calculate `enr_days = as.numeric(enr_end - enr_start)`
- Left join to excluded_patients (preserves patients with zero enrollment)
- `coalesce(has_enrollment, FALSE)` for patients without enrollment records
- Store in `patient_enrollment`
- Log patients with/without enrollment

**Section 4: TUMOR_REGISTRY exploration** (per D-02)
- Check all three TR tables for ANY records (not just HL histology matches) for the 19 patients
- For TR1: `semi_join(excluded_patients, by = "ID")` then `distinct(ID) %>% mutate(tr_table = "TR1")`
- Same for TR2, TR3 with null checks: `if (!is.null(pcornet$TUMOR_REGISTRY1)) { ... }`
- Bind all TR results together, group by ID, summarise: `has_tr_record = TRUE`, `tr_tables = paste(unique(tr_table), collapse = "+")`
- If no TR tables exist or no records found, create empty tibble: `tibble(ID = character(), has_tr_record = logical(), tr_tables = character())`
- Log per-table counts and total

**Section 5: Gap classification** (per D-05, Claude's discretion on categories)
- Build `patient_summary` by starting from `excluded_patients %>% select(ID, SOURCE, HL_SOURCE)`
- Left join diagnosis count: `neither_dx %>% count(ID, name = "n_diagnoses")`
- Left join lymphoma code count: `neither_lymphoma %>% count(ID, name = "n_lymphoma_codes")`
- Left join enrollment info from Section 3
- Left join TR info from Section 4
- Coalesce all count/boolean columns to 0/FALSE for NAs
- Apply gap classification via `case_when()`:
  - `n_diagnoses == 0 & !has_enrollment` -> `"Phantom record (no dx, no enrollment)"`
  - `n_diagnoses == 0 & has_enrollment` -> `"Coding gap (enrollment exists, zero dx)"`
  - `n_lymphoma_codes > 0` -> `"Has lymphoma/cancer codes (not HL-specific)"`
  - `n_diagnoses > 0 & !has_tr_record` -> `"Non-HL diagnoses only (no TR backup)"`
  - `has_tr_record & n_diagnoses == 0` -> `"TR record exists but no dx codes"`
  - `has_tr_record & n_diagnoses > 0` -> `"Non-HL dx + non-HL TR"`
  - `TRUE` -> `"Uncategorized (requires manual review)"`
- Final select: ID, SOURCE, HL_SOURCE, n_diagnoses, n_lymphoma_codes, has_enrollment, enr_days, n_enrollment_records, has_tr_record, tr_tables, gap_classification

**Section 6: Console summary** (per D-08)
- Print structured summary using `message()` and `glue()` following the pattern from 04_build_cohort.R Section 8
- Include sections:
  - Header with `strrep("=", 60)` and title "NEITHER PATIENTS GAP ANALYSIS SUMMARY"
  - Total excluded patients count and sites
  - "--- Diagnosis Coverage ---": patients with ANY dx codes, patients with ZERO dx codes, total dx records, lymphoma/cancer codes found
  - "--- Enrollment Coverage ---": patients with/without enrollment
  - "--- TUMOR_REGISTRY Coverage ---": patients with/without TR records
  - "--- Gap Classification ---": loop through `patient_summary %>% count(gap_classification)` printing each
  - "--- Gap Classification by Site ---" (per D-03): print `tabyl(patient_summary, SOURCE, gap_classification) %>% adorn_totals(c("row", "col"))`
  - If any lymphoma codes found, print the specific codes: `message("Lymphoma/cancer codes found:")` followed by distinct DX values
  - Recommendation line: if `nrow(neither_lymphoma) == 0`, print "No lymphoma/cancer codes found in Neither patients. Gap is a data quality limitation, not a code list issue. No pipeline changes recommended." Otherwise print "Lymphoma/cancer codes found -- review neither_lymphoma_codes.csv to determine if HL identification expansion is warranted."
  - Footer with `strrep("=", 60)`

**Section 7: CSV outputs** (per D-07)
- Create directory: `dir.create(file.path(CONFIG$output_dir, "diagnostics"), showWarnings = FALSE, recursive = TRUE)`
- Write `neither_all_diagnoses.csv`: all rows from `neither_dx`
- Write `neither_lymphoma_codes.csv`: all rows from `neither_lymphoma` (may be 0 rows -- that's informative)
- Write `neither_patient_summary.csv`: all rows from `patient_summary`
- Log each write with row count: `message(glue("Wrote {nrow(df)} rows to {filename}"))`
- Final message: `message(glue("\nAll outputs saved to {file.path(CONFIG$output_dir, 'diagnostics')}/"))`
