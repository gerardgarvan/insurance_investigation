## Phase Details

### Phase 45: Radiation CPT Audit
**Goal**: The radiation CPT range 70010-79999 is documented, every code in HL patient data is classified as imaging or treatment, and proton therapy codes are captured in config.
**Depends on**: Phase 44 (treatment episode work may touch R/00_config.R)
**Requirements**: RADCPT-01, RADCPT-02, RADCPT-03
**Success Criteria** (what must be TRUE):
  1. User can read a classification table of CPT 70010-79999 sub-ranges with AMA/CMS citations identifying which sub-ranges are diagnostic imaging, treatment planning, treatment delivery, and proton therapy
  2. User can see every CPT code from the 70010-79999 range that appears in HL patient PROCEDURES data, with each code labeled imaging or treatment
  3. Proton therapy codes 77520-77525 are present in TREATMENT_CODES$radiation_cpt in R/00_config.R with citation comments
  4. The classification table is written to a file (xlsx or csv) reviewable outside R
**Plans**: TBD

### Phase 46: Treatment Code Cross-Reference & Triggering Codes
**Goal**: Users can see which codes are in TreatmentVariables_2024.07.17.docx but not in config (and vice versa), and each treatment episode row in the CSV output shows which code(s) triggered it.
**Depends on**: Phase 45 (config must reflect corrected radiation codes before gap comparison)
**Requirements**: TXREF-01, TXREF-02
**Success Criteria** (what must be TRUE):
  1. User can see a two-way gap report: codes in TreatmentVariables_2024.07.17.docx not in TREATMENT_CODES, and codes in TREATMENT_CODES not in the docx
  2. The gap report is organized by treatment category (chemo, radiation, SCT, immunotherapy) so the user can scan by type
  3. The treatment episode CSV output includes a triggering_codes column showing which specific code(s) caused each episode's start date
  4. Existing episode output columns and row counts are unchanged (triggering_codes is additive)
**Plans**: TBD

### Phase 47: Cancer Site Frequency
**Goal**: Users can see how all 42 cancer site categories from CancerSiteCategories.xlsx distribute across the full PCORnet extract, and can email the result to collaborators.
**Depends on**: Nothing (fully independent of radiation audit and treatment code work)
**Requirements**: CSITE-01, CSITE-02
**Success Criteria** (what must be TRUE):
  1. User can see patient counts and encounter counts for all 42 cancer site categories from CancerSiteCategories.xlsx across all patients in the PCORnet extract (not just the HL cohort)
  2. Every ICD code in CancerSiteCategories.xlsx is accounted for — categories with zero matches show 0, not absent
  3. Output is a styled xlsx file the user can email to the group without further formatting
