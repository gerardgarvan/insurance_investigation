# Requirements: PCORnet Payer Variable Investigation (R Pipeline)

**Defined:** 2026-05-15
**Core Value:** A working cohort filter chain that reads like a clinical protocol — with logged attrition at every step and clear payer-stratified visualizations showing how patients flow from enrollment through diagnosis to treatment.

## v1.6 Requirements

Requirements for milestone v1.6: Treatment Code Validation & Cancer Site Analysis.

### Radiation CPT Audit

- [ ] **RADCPT-01**: User can see a classification table of CPT 70010-79999 sub-ranges showing which are diagnostic imaging, treatment planning, treatment delivery, and proton therapy, with AMA/CMS citations
- [ ] **RADCPT-02**: User can see which codes from the 70010-79999 range actually appear in HL patient PROCEDURES data, with each code classified as imaging vs treatment
- [ ] **RADCPT-03**: Proton therapy codes 77520-77525 are added to TREATMENT_CODES$radiation_cpt in R/00_config.R with citation comments

### Treatment Code Cross-Reference

- [ ] **TXREF-01**: User can see a two-way gap report comparing TreatmentVariables_2024.07.17.docx code lists against current TREATMENT_CODES in R/00_config.R (codes in doc but not config, and codes in config but not doc)
- [ ] **TXREF-02**: User can see which code(s) triggered each treatment episode's start date in the episode CSV output (new triggering_codes column)

### Cancer Site Frequency

- [ ] **CSITE-01**: User can see a frequency table of all 42 cancer site categories from CancerSiteCategories.xlsx across all patients in the PCORnet extract (not just HL cohort), with patient counts and encounter counts per site
- [ ] **CSITE-02**: Cancer site frequency output is a styled xlsx ready to email to the group

## Future Requirements

None deferred from v1.6.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Rewriting existing treatment detection to narrow radiation range | v1.6 audits and documents; any detection changes would be a separate milestone |
| Surgery code audit | TreatmentVariables doc references ComprehensiveSurgeryCodes.xlsx — separate effort |
| Ancillary therapy code audit | Separate SEER*Rx category — not in meeting notes scope |
| ICD-O-3 morphology matching | CancerSiteCategories.xlsx has ICD-O-3 topography codes; morphology (histology) matching is a different analysis |
