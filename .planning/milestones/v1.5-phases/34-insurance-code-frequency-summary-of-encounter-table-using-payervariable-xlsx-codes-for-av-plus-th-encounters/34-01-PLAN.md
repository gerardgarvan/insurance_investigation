<objective>
Create a standalone R diagnostic script that produces frequency tables of raw PAYER_TYPE_PRIMARY and PAYER_TYPE_SECONDARY codes in AV+TH encounters, cross-referenced against PayerVariable.xlsx to show each code's description and mapped category.

Purpose: Give the user a clear view of which payer codes actually appear in the encounter data and how they map to the xlsx's own category scheme (Medicare, Medicaid, Other govt, Private, Uninsured, Other, Impute), independent of the R pipeline's PAYER_MAPPING. Codes present in data but absent from the xlsx are flagged.

Output:
- R/35_payer_code_frequency_av_th.R (standalone script)
- 3 CSV files in output/tables/: payer_primary_code_freq_av_th.csv, payer_secondary_code_freq_av_th.csv, payer_category_summary_av_th.csv
- Console summary on HiPerGator
</objective>

Create standalone diagnostic script R/35_payer_code_frequency_av_th.R following the Phase 33 standalone script pattern (R/33_multi_source_overlap_av_th.R as structural template). The script:

**Header block:** Standard header comment with Phase 34, requirements (PAYFREQ-01 through PAYFREQ-06), purpose, output files, usage, dependencies.

**Section 0 - Setup:**
- `source("R/00_config.R")` then `library(dplyr)`, `library(glue)`, `library(readr)`, `library(readxl)` (per D-08)
- Define `PAYER_XLSX_PATH <- "PayerVariable.xlsx"` as a config constant at top of script (per D-09)
- Conditional load: `if (!USE_DUCKDB && !exists("pcornet")) source("R/01_load_pcornet.R")` and DuckDB connection open if needed (same pattern as R/33 lines 46-50)
- Banner message: "PAYER CODE FREQUENCY SUMMARY (AV+TH ONLY)"

**Section 1 - Load PayerVariable.xlsx:**
- `payer_lookup <- readxl::read_excel(PAYER_XLSX_PATH, sheet = "Sheet2")` (per D-08)
- Rename columns for R-friendliness: rename "Value In Data" to `code`, "What old value means" to `description`, "New Value" to `category`
- Trim whitespace from all three columns: `mutate(across(everything(), ~trimws(as.character(.))))`
- Handle the last row where code is "." and description is NA: keep it in the lookup as-is (it represents missing/blank data)
- Log: number of rows loaded, unique categories found, preview of first 5 and last 3 rows

**Section 2 - Load and filter ENCOUNTER table:**
- `enc <- get_pcornet_table("ENCOUNTER") %>% materialize() %>% filter(ENC_TYPE %in% c("AV", "TH"))` (per D-10, materialize-early pattern)
- Log total AV+TH encounters, unique patients
- Extract the two payer columns: `PAYER_TYPE_PRIMARY` and `PAYER_TYPE_SECONDARY`
- Convert payer columns to character if not already (DuckDB may return them as factor or different type)

**Section 3 - PAYER_TYPE_PRIMARY frequency table (per D-01, D-02):**
- Count frequency of each distinct `PAYER_TYPE_PRIMARY` value (including NA, empty string "")
- For NA values: represent as `"<NA>"` in the code column for clarity
- For empty strings: represent as `"<EMPTY>"` in the code column
- Left join to payer_lookup on code to get description and category columns
- For codes NOT found in xlsx: set description = "NOT IN XLSX" and category = "NOT IN XLSX" (per D-03)
- Add columns: `n` (count), `pct` (percentage of total, rounded to 2 decimal places)
- Sort by `n` descending (Claude's discretion: frequency-based sort is most useful)
- Store as `primary_freq`

**Section 4 - PAYER_TYPE_SECONDARY frequency table (per D-01, D-02):**
- Same logic as Section 3 but for `PAYER_TYPE_SECONDARY`
- Store as `secondary_freq`

**Section 5 - Category-level summary CSV (per D-07):**
- For BOTH primary and secondary fields:
  - Group the frequency tables by `category` (the xlsx column C value)
  - Sum the encounter counts within each category
  - Compute percentage of total
- Create a combined summary with columns: `field` ("PRIMARY" or "SECONDARY"), `category`, `n`, `pct`
- Sort by field then descending n within each field
- Store as `category_summary`

**Section 6 - Write CSV outputs:**
- `output_dir <- file.path(CONFIG$output_dir, "tables")`
- `dir.create(output_dir, showWarnings = FALSE, recursive = TRUE)`
- CSV 1: `payer_primary_code_freq_av_th.csv` -- columns: code, description, category, n, pct -- from `primary_freq`
- CSV 2: `payer_secondary_code_freq_av_th.csv` -- columns: code, description, category, n, pct -- from `secondary_freq`
- CSV 3: `payer_category_summary_av_th.csv` -- columns: field, category, n, pct -- from `category_summary`
- Use `readr::write_csv()` for all outputs
- Log each file written with row count

**Section 7 - Console summary:**
- Print separator banner
- Total AV+TH encounters analyzed
- For PRIMARY:
  - Number of distinct raw codes found
  - Number of codes NOT in xlsx
  - List codes NOT in xlsx (with their counts) if any
  - Top 10 codes by frequency (code | description | category | count | pct)
  - Category-level breakdown (category | count | pct)
- For SECONDARY:
  - Same structure as PRIMARY
- List of CSV files written
- End banner

**Key implementation notes:**
- Handle NA and empty string payer values explicitly -- these are real and important data points
- The `.` code in xlsx represents missing data; in R data it may appear as actual "." or as NA
- When joining, ensure the join column types match (both character)
- Do NOT use the R pipeline's PAYER_MAPPING at all (per D-04, D-05)
- No per-site or per-year breakdown (per D-06: overall frequencies only)
- No HIPAA suppression needed (these are aggregate code counts across all encounters, not patient-level)
