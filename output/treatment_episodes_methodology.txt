================================================================================
TREATMENT EPISODE METHODOLOGY
How the *_episodes.csv files are produced
================================================================================

Generated from codebase analysis of:
  - R/00_config.R        (treatment code definitions)
  - R/43_treatment_durations.R  (date extraction and episode logic)
  - R/44_treatment_episodes.R   (episode-level output)


================================================================================
1. OUTPUT FILES
================================================================================

Four CSV files are produced, one per treatment type:

  chemotherapy_episodes.csv
  radiation_episodes.csv
  sct_episodes.csv
  immunotherapy_episodes.csv

Each row represents one treatment episode for one patient:

  patient_id              - Patient identifier
  episode_number          - Sequential episode number per patient (1, 2, 3, ...)
  episode_start           - Date of first treatment event in the episode
  episode_stop            - Date of last treatment event in the episode
  episode_length_days     - Days between first and last event (0 if single-date)
  distinct_dates_in_episode - Count of unique treatment dates within the episode
  historical_flag         - TRUE if episode_stop falls before 2012-01-01


================================================================================
2. EPISODE DEFINITION (HOW DATES ARE GROUPED INTO EPISODES)
================================================================================

All treatment dates for a patient (across all data sources) are sorted
chronologically, then grouped using a 90-day window rule:

  1. The first date starts Episode 1.
  2. Each subsequent date is compared to the START date of the current episode.
  3. If the date is >= 90 days from the episode start, a new episode begins.
  4. Otherwise the date is added to the current episode.

This is a "window from episode start" approach, not a gap between consecutive
dates. It ensures no episode ever spans more than 90 days from its first event.

Before episode assignment, dates from all data sources are stacked and
deduplicated: if a patient has a PROCEDURES record and a DIAGNOSIS record on
the same day for the same treatment type, that day counts once.


================================================================================
3. PCORnet CDM TABLES QUERIED
================================================================================

Up to 7 tables are queried depending on treatment type:

  Table               Column Used as Date     What Is Matched
  ------------------  ----------------------  --------------------------------
  PROCEDURES          PX_DATE                 CPT, HCPCS, ICD-9, ICD-10-PCS,
                                              revenue codes in PX column
  PRESCRIBING         RX_ORDER_DATE           RXNORM_CUI
                      (fallback: RX_START_DATE)
  DISPENSING          DISPENSE_DATE           RXNORM_CUI
  MED_ADMIN           MEDADMIN_START_DATE     RXNORM_CUI
  DIAGNOSIS           DX_DATE                 ICD-10-CM and ICD-9-CM codes
  ENCOUNTER           ADMIT_DATE              MS-DRG codes
  TUMOR_REGISTRY_ALL  (varies by column)      Treatment date columns

7. IMMUNOTHERAPY (CAR T-CELL) -- CODE DEFINITIONS AND SOURCES
================================================================================

Immunotherapy detection uses only 2 sources. Checkpoint inhibitors like
nivolumab and pembrolizumab are captured under CHEMOTHERAPY (their HCPCS
and RxNorm codes are in the chemo code lists). The immunotherapy category
here specifically targets CAR T-cell therapies.

  The code references DRG 018 (Chimeric Antigen Receptor T-cell
  Immunotherapy), but as of this writing the immunotherapy_drg vector

================================================================================
8. SUPPORTIVE CARE CODES (NOT USED FOR EPISODES)
================================================================================

The config also defines supportive_care_rxnorm codes (growth factors,
antiemetics, steroids, etc.). These are NOT used in episode construction.
They were classified during drug investigation (Phase 40) to separate
true treatment drugs from supportive medications.


================================================================================
9. DATA FLOW SUMMARY
================================================================================

  Raw PCORnet CSVs (loaded via 01_load_pcornet.R into DuckDB)
       |
       v
  43_treatment_durations.R
    - extract_chemo_dates()         --> queries 7 tables, 10 code sources
    - extract_radiation_dates()     --> queries 4 tables, 7 code sources
    - extract_sct_dates()           --> queries 4 tables, 8 code sources
    - extract_immunotherapy_dates() --> queries 2 tables, 2 code sources
    - stack_and_dedup()             --> deduplicates same patient+date combos
       |
       v
  44_treatment_episodes.R
    - calculate_episodes_detailed() per treatment type
      - assign_episode_ids()        --> 90-day window-from-start clustering
      - group by (patient, episode) --> aggregate start/stop/length/count
    - Add historical_flag           --> episode_stop < 2012-01-01
       |
       v
  Output:
    chemotherapy_episodes.csv
    radiation_episodes.csv
    sct_episodes.csv
    immunotherapy_episodes.csv


================================================================================
10. KEY PARAMETERS
================================================================================

  GAP_THRESHOLD = 90 days    (window from episode start to split episodes)
  HISTORICAL_CUTOFF = 2012-01-01  (episodes ending before this are flagged)
  TREATMENT_TYPES = Chemotherapy, Radiation, SCT, Immunotherapy


================================================================================
11. KNOWN LIMITATIONS
================================================================================

  1. immunotherapy_drg is referenced in code but not defined in config.
     DRG 018 (CAR T-cell Immunotherapy) is not being captured.

  2. Checkpoint inhibitors (nivolumab, pembrolizumab) appear in the
     CHEMOTHERAPY code lists, not the IMMUNOTHERAPY category. This is
     intentional -- the diagnosis codes Z51.12 / V58.12 cover both
     chemo and immunotherapy encounters, and HCPCS J-codes for these
     drugs (J9299, etc.) are classified under chemo J-codes.

  3. All chemotherapy codes are pooled. There is no distinction between
     ABVD, BEACOPP, BV+AVD, salvage ICE/DHAP, or other regimens.

  4. Tumor registry columns are queried only if they exist in the
     loaded data. Column availability varies by data extract.
